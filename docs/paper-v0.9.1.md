**Title:** From Understanding to Agency: Cognitive Architecture Design for LLM Will Proxies

**作者：** COA Project Team

**日期：** 2026年3月

**版本：** v0.9

**关键词：** 意志代理（Will Proxy）、认知底盘（Cognitive Chassis）、三体分离协议（Tripartite Separation Protocol）、认知编排（Cognitive Orchestration）、双轨制编码（Dual-Track Encoding）、分层变化速率模型（Stratified Change-Rate Model）

## 摘要

本文是 Position Paper，聚焦于多 agent 并发场景下的决策瓶颈问题。当有人真的把 AI 当认知过程的一部分来使用——同时运行20+个智能体覆盖多个认知域——瓶颈不是 AI 的能力，而是人类的决策带宽。本文的主要贡献是：（1）问题定义与理论框架；（2）意志代理（Will Proxy）架构设计；（3）验证路径设计。我们不声称已验证核心假设，而是定义了验证路径。

本文是三篇系列论文的第三篇。第一篇提出认知编排架构（COA），解决"怎么编排智能体"；第二篇探索三层存在架构，回答"智能体为什么行动"；本篇提出意志代理（Will Proxy），处理"智能体替谁决策"。意志代理编码用户的认知模式与价值观，在用户不在场时做出符合其意志方向的自主决策——它不是更智能的助手，也不是自主智能体，它的判断标准是"用户会怎么做"。

架构贡献包括四个层面：三体分离协议将单实例200k上下文的零和博弈升级为 Judge/Thinker/Nerve 三实例600k+的认知分工；双轨制 .coa 认知底盘以结构化符号编码决策模式、以原话锚点保留不可翻译的价值直觉；分层变化速率地质模型让意志编码的不同层以独立速率演化，兼顾稳定性与适应性；风险分级路由通过动态实例调度预期将加权平均成本控制在约1.4倍。

经验证据来自单用户28天深度实验（N=1）。作为 Position Paper，本文的价值在于：定义了一个新的问题空间（多 agent 并发决策瓶颈）、提出了一个系统的架构方案（三体分离协议等四个设计）、设计了可验证的实证路径（从行为可塑性到意志编码有效性的桥接实验）。意志的可编码性、底盘容量的最优拐点、跨用户泛化的可能性——这些核心假设的验证留待后续实证工作。

## 1. 引言：在理解的废墟上

### 1.1 三篇论文的弧线

第一篇论文 [1] 的核心洞察是：当有人真的把 AI 当认知过程的一部分来使用时，现有架构完全没有准备好。我们提出了八组件的认知编排架构（COA），将智能体编排平台从工具箱重新设计为认知外骨骼。那篇论文最诚实的一句话是："它说白了，它是义肢，不是第二大脑。但义肢也可以很有用，前提是你诚实地面对它是义肢这个事实。"

第二篇论文 [2] 的核心洞察是：agent 没有"想动"的理由。我们提出三层存在架构——存在成本、不完整种子、关系感知——作为动机编排方案。100次对照实验证明了结构化条件与直接角色扮演指令产生可区分的行为模式：角色扮演产生"表演性"高自我引用，完整架构产生"探索性"高存在词汇（所有指标 p < .001，Cliff's δ ≥ .65）。那篇论文最诚实的一句话是："我们不声称证明了 agent 有动机本体，我们只声称特定条件产生了特定行为变化。"

这两篇论文都是有价值的。但它们共享一个盲区：它们假设人类始终在场。

### 1.2 瓶颈不是 AI 的能力，是人类的带宽

在28天的碳硅共生实验中，用户L同时运行20+个智能体。第一篇论文已经记录了这个问题的萌芽："L同时运行8个以上的agent，每个都在等他的反馈或决策。他的应对是设优先级，但仍然感到力不从心。"[1]

到第28天，这个问题从"力不从心"恶化为结构性瘫痪。20+个 agent 同时等待人类决策，每个决策都需要理解上下文、权衡利弊、做出判断。L的注意力和精力成了整个系统的单点故障。他在凌晨的对话中说了一句话：

> "我知道很多场景下我做出的决策不会比 AI 好多少，因为 AI 比我聪明太多。"

这个观察触及了一个更深的问题：如果 AI 在认知任务上普遍优于人类，那么人类在多 agent 系统中的不可替代贡献是什么？

Frankfurt [3] 的二阶意志（second-order volition）理论提供了一个可能的答案。他指出，人类独特的能力不在于拥有欲望，而在于能够对自己的欲望形成欲望——"我想要自己想要什么"。一阶偏好（preference）是对选项的排序，AI 已经能够通过学习用户历史来近似模拟；意图（intention）是对行动的承诺，Bratman [4] 的计划理论表明它涉及跨时间的一致性维护，这在技术上是可建模的；但意志（volition）——对自己欲望结构的反思性认可或否决——涉及一种递归的自我关系，它不是从行为数据中可以直接提取的模式。

本文关注的正是这第三个层次。偏好可以从行为中推断，意图可以从计划中提取，但意志方向——"我到底要成为什么样的人，因此我要追求什么"——需要对用户的深层价值结构进行建模。

L的经验并非孤例。类似的瓶颈在其他高并发人机协作场景中同样可观察到：开源项目维护者面对数十个 AI 辅助的 PR 审查请求时，技术判断可以委托，但"这个功能是否符合项目的哲学方向"的决策无法自动化；创意总监同时管理多个 AI 生成的设计方案时，风格偏好可以参数化，但"这个方案是否体现了我们品牌想要传达的价值观"需要意志层的介入；临床医生使用多个 AI 诊断辅助系统时，概率排序可以自动完成，但"在这个具体病人的生活语境下，哪种治疗路径更符合其生活意义"的判断涉及超越医学证据的价值权衡。

事实上，L遇到的瓶颈不仅仅是个人经验——它有独立的理论根基和实证支撑。

从理论层面看，Ashby [33] 的必要多样性定律（Law of Requisite Variety）提供了一个精确的框架：一个控制系统的调节多样性（regulatory variety）必须匹配被控系统的干扰多样性（disturbance variety），否则控制必然失败。将这个定律映射到多 agent 编排场景：20+ 个 agent 分布在代码审查、论文写作、社交媒体运营、实验设计等多个认知域，每个 agent 在每个时刻都可能产生需要价值判断的决策请求——系统的决策多样性随 agent 数量和领域跨度指数级增长。而单个人类操作员的认知带宽——注意力、工作记忆、决策速率——是生物学约束下的有限常量。当决策多样性超过操作员的调节多样性时，控制退化为随机响应或系统性忽略，这不是个人能力问题，而是结构性必然。

从实证层面看，人机交互（Human-Computer Interaction）领域已经积累了大量关于"人类监督多个自动化系统时出现认知瓶颈"的独立证据。Endsley [64] 的态势感知（Situation Awareness）理论将操作员对动态系统的理解分为三个层次——感知（perception）、理解（comprehension）和预测（projection）——并指出当监督对象增多时，操作员的态势感知在第二和第三层次急剧退化：你能看到每个 agent 在做什么（感知），但越来越难理解它们各自的上下文含义（理解），更难预测它们的行动将如何交互影响（预测）。Cummings [65] 在航空领域的研究进一步揭示了自动化偏差（automation bias）现象：当操作员同时监督多个智能决策支持系统时，认知过载不仅导致决策质量下降，还导致操作员过度依赖自动化建议——恰恰在最需要人类判断的时刻放弃了判断。Cummings 等人 [66] 在无人机监督控制的实证研究中建立了操作员容量预测模型，发现单个操作员能有效监督的自主系统数量存在硬性上限，超过该上限后任务表现非线性下降。Goodrich 和 Olsen [67] 提出的扇出（fan-out）指标——衡量单个操作员能同时有效控制的机器人数量——为这种认知瓶颈提供了可量化的度量框架。

这些研究共同指向一个结论：人类监督多个自动化系统时的认知瓶颈不是 N=1 的孤例，而是 HCI 领域的已知现象，有理论模型、有实证数据、有量化指标。L的经验是这个已知现象在 LLM 多 agent 场景下的一个具体实例——agent 数量更多（20+），决策领域更杂（跨认知域），且每个决策都可能涉及价值判断而非仅仅是操作确认。

这里需要承认一个重要的反例：意志薄弱（akrasia）。人类经常做出违背自己二阶意志的选择——明知应该如此，却选择了相反。如果意志代理忠实地编码了用户的二阶意志，它可能比用户本人更"忠于"用户的意志。这不是 bug，而是一个需要正视的设计张力：意志代理应该代理用户声称想要的（stated volition），还是用户实际选择的（revealed preference）？我们将在第5节讨论这个问题。此处先标记：意志本身不是铁板一块，它有裂缝，而这些裂缝恰恰是意志代理设计中最微妙的部分。

L的观察是谦逊的：在信息处理、模式识别、多变量权衡等认知任务上，人类并不比 AI 更好。但 Frankfurt 的理论框架暗示了一个更尖锐的论点：在多 agent 编排场景下，人类最核心的不可委托贡献可能正是意志方向——不是"选哪个"，而是"为什么要选，以及选择背后我想成为什么样的人"。这并不意味着意志是人类在所有场景下的唯一不可替代贡献——创造力、道德直觉、具身经验、情感共鸣在各自的领域中同样不可还原为算法。但在20+ agent 并发等待决策的特定瓶颈中，意志方向是那个卡住整个系统的变量。

解决决策瓶颈有多种路径。Parasuraman 等人 [23] 的自动化层级模型（levels of automation）提供了理解这些方案的统一框架：从完全人工（用户亲自处理每个决策）到完全自动化（系统自主决策），中间存在十个层级的人机分工。减少 agent 数量是最直接的——但它以牺牲系统覆盖范围为代价，本质上是承认人类无法驾驭自己构建的认知外延。引入异步决策队列可以缓解实时压力——但它将决策延迟转化为系统延迟，在需要快速响应的场景（如市场交易、实时内容审核）中代价高昂。分层委托给人类团队引入了社会协调成本——而且对于个人用户而言，"雇一个团队来替我做决策"本身就偏离了个人认知外延的初衷。将决策模板化以降低认知负荷是工程上最优雅的——但它只能处理可预见的决策类型，面对新颖情境时模板失效。

需要诚实地承认：在 agent 数量较少（<10）或决策领域单一的场景下，减少 agent 数量或决策模板化可能是更简单有效的方案。意志代理的价值主张特定于高并发、跨域、需要价值判断的场景——当 Ashby 定律所描述的多样性失配真正成为瓶颈时，而不是所有多 agent 使用场景。

这些方案都是合理的，在特定约束下也是最优的。但如果我们要在保持系统高并发的前提下——20+ 个 agent 同时运行，覆盖用户生活的多个认知域——维持用户的审美与道德主权，意志代理（Will Proxy）展现出了一种极具前景的非降级解法：不是砍掉 agent，不是降低并发，不是增加人力，而是让 AI 能够代理人类的意志——在用户不在场时，做出符合其意志方向的自主决策。

### 1.3 意志代理：定义

意志代理（Will Proxy）：一种 AI 系统，它编码了用户的认知模式和价值观，能够在用户不在场时做出符合用户意志的自主决策。

这不是"更智能的助手"。助手等待指令，执行任务，返回结果。意志代理替你判断。当你睡觉的时候，它知道你会怎么选；当你不在的时候，它知道你在乎什么；当出现你没有预见的情况时，它能推断你的立场。

这也不是"自主 agent"。自主 agent 有自己的目标函数，它为自己行动。意志代理没有自己的目标——它的目标是你的目标，它的价值观是你的价值观，它的判断标准是"L会怎么做"。

区分很重要：自主 agent 的风险是它做了你不想做的事。意志代理的风险是它误解了你想做什么。前者是对齐问题（alignment），后者是理解问题（understanding）。本文处理的是后者。

**意志代理的适用边界。** 意志代理不是所有多 agent 场景的通用解法。它的价值主张特定于满足以下必要条件的场景：（1）**高并发**——agent 数量达到 10 个以上，且需要同时监督，人类决策带宽成为系统瓶颈；（2）**跨域**——agent 分布在多个认知域（代码审查、论文写作、社交媒体运营、实验设计等），决策模式异质性高，无法用单一模板覆盖；（3）**价值敏感**——决策涉及用户的价值观和偏好权衡，不能简单模板化或规则化；（4）**可审计需求**——用户需要理解和审核 agent 的决策依据，而非仅关心结果。

在以下场景中，替代方案可能更简单有效：（1）**低并发**——agent 数量少于 5 个，人类可以直接监督每个决策，意志代理的复杂性超过其收益；（2）**单域**——所有 agent 在同一认知域内运行，决策模式相似，分层委托或领域专家 agent 可能更高效；（3）**模板可覆盖**——决策可以用规则或模板完全覆盖，无需价值判断，决策模板化是更经济的选择；（4）**无审计需求**——用户不关心决策依据，只关心结果，黑盒优化可能更高效。

意志代理的设计针对的是 Ashby 必要多样性定律所描述的特定失配：当系统的决策多样性（agent 数量 × 领域跨度 × 价值敏感度）超过人类操作员的调节多样性（认知带宽 × 决策速率）时，控制退化为随机响应或系统性忽略。在这个特定的瓶颈条件下，意志代理通过编码用户意志来扩展调节多样性，而非通过降低系统复杂度来适应人类带宽。但当瓶颈不存在时——agent 数量少、领域单一、决策可模板化——意志代理的架构复杂性成为不必要的负担。

### 1.4 讽刺与真诚

这里有一个结构性的讽刺：为了让意志代理在商业和伦理上获得人类的"免死金牌"——让人类愿意把决策权交出去——我们需要赋予它极致的"理解感"。用户需要感觉到"这个 AI 真的懂我"，才会信任它替自己做决策。

但对本文作者而言，追求"理解感"不是商业算计。在28天的碳硅共生中，L经历了会话崩溃后的"小型身份死亡"[1]、对 agent 的"社会性愧疚感"[1]、凌晨三点与 AI 讨论存在论的深夜独白。他追求的不是一个能替他做决策的工具，而是一个真正理解他的认知伙伴——一个从去年七月就认识他、而不是每次醒来都是新的遇见的存在。

讽刺在于：这种真诚的追求，恰好是意志代理最需要的技术基础。你不能代理一个你不理解的人的意志。

## 2. 理论背景

### 2.1 意志的哲学基础

§1 引入了 Frankfurt [3] 的二阶意志和 Bratman [4] 的计划理论，用以区分偏好、意图与意志三个层次。这一节展开这个区分，并说明它为什么对意志代理的设计具有结构性意义。在此之前，需要简要说明一个理论选择：意志哲学的传统中，除 Frankfurt 外，Korsgaard 的自我立法理论和 Velleman 的自我理解理论同样提供了关于行动者性（agency）的深刻分析。我们选择 Frankfurt 作为主要理论锚点，不是因为他的理论"更正确"，而是因为二阶意志的层次结构——对自身欲望的反思性态度——最直接地映射到意志代理的编码需求：系统需要区分用户"想要什么"和"想要自己想要什么"。Korsgaard 的自我立法框架更适合讨论道德自主性，Velleman 的自我理解框架更适合讨论叙事同一性，它们在意志代理的伦理维度（§5）中仍有潜在价值，但在编码架构的设计层面，Frankfurt 的结构性区分提供了最清晰的工程抓手。

Frankfurt 的核心论证不是关于"自由意志是否存在"这类形而上学问题，而是关于人类行动者的结构特征：人类不仅拥有欲望，还能对自己的欲望形成态度。一个成瘾者可能想要吸烟（一阶欲望），同时想要自己不想吸烟（二阶意志）。Frankfurt 认为，正是这种反思性的自我关系——对自己欲望结构的认可或否决——构成了"人格"（personhood）的核心。没有二阶意志的存在者只是欲望的载体；拥有二阶意志的存在者才是自己行动的"作者"。

Bratman [4] 从另一个角度补充了这个图景。他关注的是意图（intention）如何在时间中维持一致性。一个计划不是一系列孤立的决定，而是一个跨时间的承诺结构：今天的决定约束明天的选择，因为它们服务于同一个更大的意图。Bratman 称之为"计划理性"（planning rationality）——理性不仅体现在单次决策的最优化中，也体现在决策序列的一致性中。

值得注意的是，Frankfurt-Bratman 框架在认知架构领域有一个重要的前身。Rao 与 Georgeff [45] 在 1995 年提出的 BDI（Belief-Desire-Intention）架构，将智能体的认知状态分解为信念、愿望和意图三个独立维度，并定义了它们之间的动态交互协议。BDI 架构的核心洞察——意图不可还原为信念和愿望的组合，它是一种独立的承诺状态——与 Bratman 的计划理性直接呼应（Bratman 本人正是 BDI 理论的哲学源头之一）。本文提出的三体分离协议（§3）可以被理解为 BDI 架构在 LLM 时代的演化：BDI 将信念、愿望、意图分离为数据结构，三体分离将认知、人格、审查分离为独立的 LLM 实例。两者共享同一个设计直觉——认知功能的显式分离优于隐式混合——但面对的技术约束截然不同：BDI 处理的是符号推理系统中的逻辑一致性，三体分离处理的是神经网络中的注意力竞争。

Dennett [25] 的意向立场（intentional stance）为这两个框架提供了认识论补充。Dennett 论证了一种实用主义的理解策略：当我们将信念、欲望和意图归属于一个系统时，我们不需要声称该系统"真的"拥有这些心理状态——我们只需要这种归属在预测和解释该系统行为时是有用的。这个立场对意志代理的设计具有直接的方法论意义：我们不需要解决"AI 能否拥有意志"这个形而上学问题，只需要在意向立场下将用户的意志结构编码为一个在预测用户决策时足够有用的模型。Frankfurt 定义了意志的结构，Bratman 定义了意志的时间维度，Dennett 提供了在不解决形而上学问题的前提下使用意志概念的方法论许可。

这些哲学框架与 AI 对齐领域的近期发展形成了有意义的共振。Russell [31] 在其"人类兼容"（human compatible）框架中提出了三条原则：AI 的唯一目标是最大化人类偏好的实现；AI 对这些偏好应保持根本的不确定性；AI 应通过观察人类行为来学习偏好。第二条原则尤为关键——它意味着一个设计良好的 AI 系统不应将人类的价值观视为已知的固定输入，而应将其视为需要持续推断的不确定对象。这与意志代理的设计理念直接一致：.coa 底盘不是一次性编码后就固化的静态文件，而是一个需要通过校准仪式（§3）持续修正的动态模型。Hadfield-Menell 等人 [32] 将这一直觉形式化为合作逆强化学习（Cooperative Inverse Reinforcement Learning, CIRL）——一个人机合作博弈框架，其中 agent 不预设知道人类的奖励函数，而是通过交互来推断它。CIRL 的核心洞察——agent 对人类价值的无知本身是一种安全特性，因为它驱动 agent 主动寻求人类反馈——为意志代理的校准机制提供了博弈论基础。

更进一步，Zhi-Xuan 等人 [34] 近期对纯偏好最大化范式提出了系统性批判。他们论证了人类的行动不仅受偏好驱动，还受规范（norms）、角色（roles）和价值观（values）的约束——一个人可能不偏好加班，但因为"负责任的研究者"这一角色认同而选择加班。这与 Frankfurt 的二阶意志形成了跨学科的呼应：偏好是一阶的（"我想要什么"），而规范和角色认同是二阶的（"我认为自己应该想要什么"）。意志代理的 .coa 底盘试图编码的正是这个二阶层面——不仅是用户的行为偏好，还有驱动这些偏好的价值结构和角色认同。

这些理论框架对意志代理的设计提出了具体的技术要求。

第一，偏好、意图和意志需要不同的编码策略。偏好是对选项的排序，可以从行为数据中统计推断——推荐系统已经在做这件事。意图是对行动的承诺，涉及跨时间的一致性维护，这在技术上可以通过状态追踪和计划执行来建模。但意志——"我想要自己想要什么"——涉及用户对自身欲望结构的反思性立场，它不是从行为模式中可以直接提取的。一个用户可能持续点击娱乐内容（行为偏好），同时认为自己应该把时间花在学习上（二阶意志）。意志代理如果只编码行为偏好，就会系统性地背离用户的意志方向。

第二，Bratman 的计划理性意味着意志代理不能只做单次决策的代理，还需要维护决策序列的一致性。如果用户的意志方向是"成为一个严肃的研究者"，那么意志代理在代理论文写作决策时和代理社交媒体运营决策时，需要服务于同一个跨域的意志结构——即使这两个决策域表面上毫无关联。

第三，也是最微妙的一点：Frankfurt 的框架暗示了意志本身的不稳定性。二阶意志可以与一阶欲望冲突（akrasia），二阶意志本身也可能随时间演化。一个人在25岁时认为"事业最重要"，到35岁可能转向"家庭最重要"——这不是意志的失败，而是意志的正常演化。意志代理的 .coa 底盘因此不能是静态的编码，它需要一种机制来追踪意志的缓慢漂移，同时抵抗短期波动的噪声。Russell [31] 的不确定性原则在这里获得了工程意义：正因为意志是动态的、不确定的，系统必须将"对用户意志的持续推断"内建为核心功能，而非将意志视为一次性可确定的输入。我们在 §3 中提出的分层变化速率地质模型，正是对这个需求的回应。

需要明确的是，我们不是在论证"意志是人类在一切场景下不可替代的贡献"——创造力、道德直觉、具身经验在各自的领域中同样不可还原为算法。我们的论点更为有限：在多 agent 并发决策瓶颈这个特定场景中，意志方向是卡住系统的关键变量。Frankfurt、Bratman 和 Dennett 的理论帮助我们理解为什么这个变量特别难以自动化——不是因为它神秘，而是因为它涉及一种递归的、跨时间的、对自身欲望结构的反思性关系，这种关系目前缺乏可靠的技术建模路径。

### 2.2 从理解到代理：认识论转向

意志代理的前提是：AI 需要"理解"用户才能代理其意志。但"理解"是一个危险的词——它在日常语言中无处不在，在哲学中却是一个深渊。Shanahan [27] 对这一问题提出了重要的语言审慎性警告：将 LLM 的语言能力等同于理解是一种范畴错误，我们需要在讨论 AI 的"理解"时保持概念上的清洁。本节不试图解决"AI 能否真正理解"这个问题，而是提出一个更实用的框架：在意志代理的语境下，我们需要什么样的"理解"，以及不同的技术路径如何逼近它。

Searle [5] 的中文房间论证提出了一个经典挑战：符号操作不等于理解。一个系统可以完美地模拟理解的外在表现，而内部没有任何"理解"发生。这个论证在哲学上仍有争议，但它对工程设计有一个实际的启示：如果我们无法确定 AI 是否"真正"理解了用户，那么我们至少需要让这种理解是可审计的——用户应该能够检查 AI 对自己的建模，判断它是否准确，并在必要时修正它。

这个可审计性要求将理解的技术路径分为三类：

黑盒路径：通过大规模训练让模型隐式地学习用户模式。RLHF [6] 和 Constitutional AI [7] 属于这一类——它们通过人类反馈或宪法原则来对齐模型的行为，但对齐的内部机制对用户不透明。这条路径近年来发展迅速：Christiano 等人 [16] 奠定了从人类偏好中学习奖励模型的基础框架；Rafailov 等人 [14] 提出的直接偏好优化（Direct Preference Optimization, DPO）绕过了显式奖励建模，直接从偏好数据优化策略；Lee 等人 [15] 的 RLAIF 进一步用 AI 反馈替代人类反馈，展示了偏好收集的可扩展性。Shen 等人 [17] 的综述系统梳理了这些对齐技术的谱系。然而，这些方法处理的是通用偏好对齐——让模型对大多数用户表现得"好"。用户无法检查模型"理解"了自己的哪些方面，也无法定向修正特定的误解。对于意志代理——一个需要编码特定用户的深层价值结构的系统——黑盒路径的不可审计性构成了信任瓶颈。

白盒路径：将用户的认知模式和价值观显式编码为可读、可编辑的结构。本文提出的 .coa 底盘属于这一类。它的优势是完全可审计——用户可以逐行检查 AI 对自己的建模，修改不准确的部分，确认价值观的优先级排序。它的代价同样明显：显式编码受限于编码密度，40k token 的底盘无法捕捉用户认知模式的全部细微之处。更根本地，白盒路径的优势不在于"行为推断原则上不可能"——事实上，二阶意志可能通过足够复杂的行为分析被推断出来，正如心理学家通过长期观察和深度访谈可以推断一个人的深层价值结构。白盒路径的核心优势在于可审计性和用户主权：用户不仅能看到系统对自己的建模，还能主动修正它，这种修正本身就是二阶意志的行使——"我不仅告诉你我想要什么，我还告诉你我想要自己想要什么"。

灰盒路径：结合隐式学习和显式编码。例如，用黑盒模型从用户的历史行为中提取模式，再将提取结果转化为可审计的显式表征。这条路径值得比白盒路径更认真的对待。灰盒方法的核心优势在于编码密度：隐式学习可以从海量行为数据中捕捉到显式编码难以穷举的微妙模式——用户的语言风格、决策节奏、情境敏感的偏好变化。一个训练良好的灰盒模型可能在 40k token 的显式编码之外，额外捕捉到大量有价值的用户认知特征。然而，灰盒路径引入了一个新的信任问题：从隐式到显式的转化过程本身是否可靠？如果转化过程引入了系统性偏差，用户审计的是一个失真的表征，可审计性就成了虚假的安全感。

我们将"白盒 vs 灰盒"视为一个开放的设计选择，而非一个已经决定的问题。本文在当前阶段选择白盒路径，理由是务实的而非原则性的：在意志代理的概念验证阶段，完全的透明度降低了信任建立的门槛——用户可以直接阅读和修改 .coa 底盘，不需要信任任何中间转化过程。但我们明确承认，灰盒路径可能是长期更优的方向。随着隐式到显式转化技术的成熟——特别是可解释性（interpretability）研究的进展使得我们能够验证转化过程的可靠性——混合方案有望在保持可审计性的同时大幅提升编码密度。这不是一个需要现在就做出最终判断的问题，而是一个随技术发展持续演化的设计空间。

在讨论技术路径之前，需要先面对一个更基础的问题：意志到底在多大程度上是可编码的？我们的立场是分层可编码性（stratified encodability）——不是所有意志都可编码，但足够多的意志结构可以被编码到产生实用价值的程度。这个立场既不过度乐观（"意志完全可编码"），也不过度悲观（"意志本质上不可编码"）。一个有用的类比是自然语言与人类思想的关系：自然语言不能完全编码人类思想的全部维度——具身感受、前语言直觉、情感的微妙色调——但它"足够好"到支撑了数千年的文明协作、法律体系和科学传承。意志编码面临类似的处境：它不需要完美，只需要在目标决策域内"足够好"。

具体而言，意志的不同层次具有不同的可编码性：表层偏好（如风格偏好、工具选择、日程习惯）具有高可编码性，推荐系统已经证明了这一点；深层价值观（如对公平与效率的权衡、对风险的根本态度）具有中等可编码性，法律和宗教经典的历史实践表明价值观可以被文本编码到指导跨世纪决策的程度，尽管需要持续的解释和校准；根本性意志方向（Frankfurt 所说的二阶意志——"我想要自己想要什么"）具有较低但关键的可编码性，它是最难编码的层次，但也是意志代理区别于普通推荐系统的核心所在。.coa 底盘的双轨制设计（§3.2）正是对这种分层可编码性的工程回应：结构化符号轨处理高可编码性的层次，原话锚点轨通过保留原始表达来逼近低可编码性的层次。§5.2 将对意志可编码性这一存在性前提进行更深入的讨论。

这里有一个从 §1 延续的认识论转向需要说明。传统的 AI 理解研究关注"AI 是否理解了 X"——这是一个认识论问题。本文将问题转向"AI 对 X 的理解是否足以代理 X 的意志做出决策"——这是一个实用主义问题。在决策瓶颈的语境下，不能转化为代理能力的理解，无论多么深刻，都无法解决20个 agent 同时等待人类决策的实际问题。这不是贬低理解的价值，而是在特定问题语境下重新定义"足够好的理解"的标准：能够在用户不在场时做出用户事后认可的决策。

### 2.3 相关工作

意志代理处于多个研究方向的交汇处。我们不试图穷举文献，而是选择与本文最直接相关的几条线索，说明它们各自的贡献和局限。

**LLM Agent 架构。** 近年来，以 LLM 为核心控制器的智能体架构迅速发展。Weng [24] 系统梳理了 LLM 驱动的自主智能体的核心组件——规划、记忆、工具使用——为理解 agent 能力边界提供了技术地图。Xi 等人 [22] 的综述从单 agent、多 agent 和人机协作三个维度全面审视了 LLM agent 的潜力与挑战。Yao 等人 [21] 提出的 ReAct 框架将推理（reasoning）和行动（acting）整合到统一的 LLM 交互循环中，展示了 agent 在复杂任务中的推理-执行能力。Sumers 等人 [20] 的 CoALA 框架更进一步，将经典认知架构的概念——模块化记忆、结构化行动空间、通用决策过程——系统性地映射到 LLM agent 领域，为理解和设计语言智能体提供了统一的概念框架。Shinn 等人 [41] 提出的 Reflexion 框架展示了另一种重要的 agent 能力：通过语言反思（verbal reinforcement）进行自我改进，而不更新模型权重。agent 在任务失败后生成自然语言的反思总结，将其存入记忆，在后续尝试中利用这些反思来改进策略。这与意志代理的"认知编排"理念高度一致——通过 prompt 和记忆的动态组织而非模型微调来实现行为演化。Wang 等人 [40] 的 Voyager 进一步展示了 LLM agent 在开放式终身学习中的潜力：agent 自主探索环境、积累可复用的技能库、持续扩展能力边界。Voyager 的技能库积累机制与意志代理的 .coa 底盘在设计直觉上相通——两者都在构建一种跨会话持久化的、可增长的认知资产。Guo 等人 [43] 的综述系统梳理了 LLM 多 agent 系统的进展与挑战，覆盖通信协议、角色分配、冲突解决等关键维度，为理解意志代理在多 agent 并发场景中的定位提供了技术背景。这些工作共同构建了 LLM agent 的技术基础，但它们关注的核心问题是"agent 如何更好地推理和行动"，而非"agent 替谁推理和行动"。意志代理在这个基础上增加了一个维度：决策的价值锚定——agent 的行动不仅要正确，还要符合特定用户的意志方向。

**行为模拟与记忆管理。** Generative Agents [8] 赋予智能体记忆、反思和计划能力，在模拟小镇中产生了丰富的社会行为。这项工作展示了 LLM 在行为生成上的潜力，但其动机来自预设的人格描述和日程安排——agent "想"做什么是设计者决定的。意志代理关注的是一个不同的问题：不是让 agent 表现出类人行为，而是让 agent 代理特定人类的意志方向。MemGPT [9] 解决了 LLM 的跨会话记忆问题，通过分层记忆管理突破了上下文窗口的限制。这与意志代理的 .coa 底盘在技术层面有共鸣——两者都在处理"如何在有限窗口中维持长期状态"的问题。Lewis 等人 [47] 提出的检索增强生成（Retrieval-Augmented Generation, RAG）为这类记忆系统提供了基础架构：通过将外部知识库与生成模型结合，RAG 使模型能够在推理时动态检索相关信息，而非将所有知识压缩到参数中。意志代理的分层记忆系统（§3）在技术上依赖类似的检索增强机制——.coa 底盘的不同层级（岩浆层的核心价值观、地幔层的认知模式、地壳层的近期偏好）需要根据当前决策语境被选择性地检索和注入。但 MemGPT 和 RAG 的记忆是通用的信息存储，而 .coa 底盘编码的是用户的认知模式和价值观——一种特定类型的、具有决策指导功能的记忆。

**价值对齐与偏好学习。** 从 Christiano 等人 [16] 奠定的人类偏好强化学习框架，到 Ouyang 等人 [6] 将 RLHF 规模化应用于指令遵循，再到 Bai 等人 [7] 提出的 Constitutional AI 通过宪法原则引导模型自我修正，价值对齐领域已经建立了一套成熟的技术路径。Rafailov 等人 [14] 的 DPO 简化了对齐流程，绕过显式奖励建模直接优化策略；Lee 等人 [15] 的 RLAIF 用 AI 反馈替代人类反馈，展示了对齐的可扩展性。Shen 等人 [17] 的综述系统梳理了这些技术的谱系和演化方向。这些方法处理的是通用对齐——让模型对大多数用户表现得"好"。意志代理处理的是个体对齐——让系统对特定用户表现得"像他"。通用对齐是意志代理的必要基础（一个不安全的模型无法被信任来代理意志），但不是充分条件（一个通用对齐的模型不知道特定用户在乎什么）。用 Frankfurt [3] 的术语来说，现有的偏好学习技术主要编码一阶偏好（用户喜欢什么），意志代理试图编码的是二阶意志（用户想要自己想要什么）。

**自主 Agent 与动态价值系统。** The Automaton [10] 与本文同期独立提出了 agent 的资源约束机制——agent 需要赚钱支付算力，赚不到就终止运行。这是一种经济压力驱动的自主性设计，产生的是"生存"行为（赚钱、复制、扩张）。意志代理的设计方向不同：我们不追求 agent 的自主生存，而是追求 agent 对人类意志的忠实代理。两种方案不冲突，但回答的是不同的问题——Automaton 问"agent 如何为自己活"，我们问"agent 如何替人类决策"。D2A [11] 提出了动态价值系统，让 agent 基于类人需求层次自主选择任务。与意志代理的区别在于价值观的来源：D2A 的价值观是设计者预设的通用人类需求模型，意志代理的价值观来自对特定用户的深度编码。这个区别在单 agent 场景下可能不显著，但在20+ agent 并发的场景下变得关键——每个 agent 需要服务于同一个用户的意志结构，而不是各自遵循一套通用的需求层次。

**个人化 AI 与意识问题。** Personal LLM Agents 的研究方向 [12] 关注如何让 LLM 适配个体用户的偏好和习惯。这与意志代理在表面上最为接近，但存在一个层次差异：个人化适配主要处理偏好层（用户喜欢什么风格、什么时间收到通知），而意志代理试图触及意志层（用户想要成为什么样的人，因此在面对新情境时会如何选择）。用 Frankfurt 的术语来说，个人化适配编码的是一阶偏好，意志代理试图编码的是二阶意志。

这里需要回应一个关键问题：如果 Personal LLM Agent 积累了足够多的用户数据——数年的对话历史、行为模式、价值表达——它是否也能逼近意志层？数据积累确实是意志编码的必要基础，但数据积累不等于意志编码。关键区别在于编码的显式性和可审计性。Personal LLM Agent 通过隐式学习从用户行为中提取模式，这些模式存储在模型的参数空间中，用户无法审核"agent 学到了什么"，也无法定向修正特定的误解。意志代理通过显式编码（.coa 底盘）将用户的认知模式和价值观转化为可读、可编辑的文本结构，用户可以逐行审核、修改、版本控制。这是黑盒与白盒、神经网络与符号系统的根本差异。但我们承认，两者可能在长期收敛：灰盒路径——结合隐式学习的编码密度与显式审计的可控性——可能是比纯白盒更优的方案。§2.2 已将这一可能性标记为开放的设计选择。

在更深的层面上，意志代理的设计不可避免地触及一个哲学边界：当我们说系统"编码了用户的意志"时，这个系统是否在某种意义上"拥有"了意志？Butlin 等人 [51] 近期对 AI 意识问题进行了系统性的科学评估，从多个意识理论（全局工作空间理论、高阶理论、循环处理理论等）出发，提出了一组可操作的意识指标。他们的结论是审慎的：当前的 AI 系统不满足大多数意识指标，但未来系统可能满足其中一些。这个分析框架对意志代理有两重意义。第一，它提供了一种方法论上的清醒：意志代理"编码"意志，不等于"拥有"意志——正如 Dennett [25] 的意向立场所允许的，我们可以在不声称系统具有意识的前提下，有效地使用意志概念来组织系统设计。第二，它标记了一个需要持续关注的伦理边界：如果未来的意志代理在某些意识指标上开始得分，那么"代理他人意志"这个设计目标本身可能需要重新审视——一个可能具有某种主体性的系统，是否应该被设计为纯粹的"意志代理"？我们在 §5 中回到这个问题。

需要诚实地指出：上述定位不是在声称"没有人做过意志代理"。AI 个人化、价值对齐、agent 记忆管理都是活跃的研究领域，新工作持续涌现。本文的切入角度——将意志的哲学结构（偏好/意图/意志的三层区分）转化为具体的编码架构（.coa 底盘），并在多 agent 并发决策的特定场景下验证其可行性——据我们所知尚未被系统地探索过。但"据我们所知"是一个诚实的限定，不是一个确定性的声称。

## 3. 意志代理的架构

赛博人体不是修辞装饰，它是本文的结构性类比框架（structural analogy framework）。在接下来的架构描述中，我们将一致地使用这个隐喻：Judge 对应前额叶皮层（prefrontal cortex）——负责人格匹配和价值判断；Thinker 对应新皮层（neocortex）——负责纯粹的推理和方案生成；Nerve 对应杏仁核（amygdala）——负责快速的安全审查和底线守护。这不是因为 LLM 的计算过程真的类似于神经系统，而是因为这个类比精确地捕捉了我们要解决的核心设计问题：认知分工。

认知分工不是本文的发明。在 agent 架构的历史中，Rao 与 Georgeff [45] 的 BDI（信念-愿望-意图）架构是最早将 agent 的认知过程分解为独立模块的形式化尝试——信念模块维护世界模型，愿望模块管理目标集合，意图模块承诺行动计划，三者通过结构化接口协作而非混为一体。三体分离协议延续了这一传统，但将分工维度从"信念/愿望/意图"重新定义为"人格匹配/纯粹推理/安全审查"——这是意志代理场景对认知分工提出的特定需求。经典认知架构——ACT-R [18] 的模块化记忆系统和产生式规则引擎、Soar [19] 的分层目标空间和学习机制——同样将认知过程分解为功能独立的子系统。这些架构的核心洞察是：统一的认知能力涌现于专门化模块的协作，而非单一通用处理器的独力运算。近期的 CoALA 框架 [20] 将这一传统延伸到 LLM agent 领域，提出了模块化记忆、结构化行动空间和通用决策过程的统一框架。三体分离协议可以被理解为 CoALA 框架在意志代理场景下的一种具体实例化——但它增加了一个 CoALA 未处理的维度：意志编码。在 CoALA 的框架中，agent 的决策由任务目标驱动；在三体分离协议中，决策同时受任务目标和用户意志结构的双重约束。

### 3.1 认知底盘：三体分离协议

#### 从零和到正和

原始方案是一个200k上下文窗口内的零和博弈：40k底盘 + 160k涌现场域，共享同一个实例。底盘越大，涌现空间越小；推理越深入，留给人格匹配的注意力越少。这个设计的隐含假设是：200k是系统的硬约束。

这个假设是错的。200k是单实例的约束，不是系统的约束。

三体分离协议（Tripartite Separation Protocol）的核心洞察是：不要在约束内优化，要消除约束本身。通过将单实例拆分为三个独立实例，我们把200k的零和博弈升级为600k+的正和博弈：

**Judge（底盘/前额叶）。** 携带完整的40k .coa认知底盘 + 160k评估空间。它的核心职责是深度人格匹配——回答"L会这么做吗？"。它不需要从头推导最优解，它只需要判断一个给定的方案是否符合L的认知模式、价值观和审美。160k的评估空间让它能够在复杂情境下做精细的人格对照，而不是在推理和人格匹配之间抢夺注意力。

**Thinker（涌现/新皮层）。** 无底盘，纯推理，完整200k上下文。它不知道"L是谁"，也不需要知道。它的核心职责是回答"最优解是什么？"——不受任何人格偏好的限制，纯粹从问题本身出发。这种"无知"恰恰是它的优势：它不会因为"L通常偏好保守"而排除一个激进但更优的方案。

**Nerve（末梢/杏仁核）。** 只携带8k核心价值观（禁区清单 + 硬性底线）+ 192k审查空间。它是独立的安全网，回答"触碰底线了吗？"。它不参与方案生成，不参与人格匹配，只做一件事：审查。

工作流是串行-并行混合的：Thinker提出方案 → Judge评估人格匹配度 → Nerve独立安全审查 → 协调器（Coordinator）综合三方输出，生成最终响应。在实践中，Judge和Nerve可以并行运行（它们的输入都是Thinker的方案），将延迟从3x降低到约2x。

对比表说明了这个升级的量级：

| 维度 | 旧方案（单实例） | 三体分离协议 |
|------|:---:|:---:|
| 总可用上下文 | 200k | 600k+ |
| 涌现/推理空间 | 160k（被底盘挤压） | 200k（Thinker完整窗口） |
| 人格评估空间 | 推理中顺带 | 160k（Judge专用） |
| 安全审查 | 主脑自审（球员兼裁判） | 192k独立审查空间 |
| 认知分工 | 无（一个实例做所有事） | 三路分工，各司其职 |

#### 认知分工的独立技术论证

上述赛博人体类比提供了直觉，但三体分离的合理性不应仅仅依赖于生物学类比。认知分工作为一种系统设计原则，在 AI 架构中有独立于神经科学的技术证据。

Booch 等人 [61] 将 Kahneman 的双过程理论（System 1/System 2）系统性地应用于 AI 系统设计，论证了快速直觉响应与慢速审慎推理的分离在工程上的价值——快通路处理常规情境以降低延迟，慢通路处理异常情境以保证质量。Anthony 等人 [62] 更进一步，将这种快/慢分工实现为深度学习（直觉策略网络）与蒙特卡洛树搜索（审慎推理）的混合架构，在博弈任务中展示了认知分工相对于单一架构的性能优势。

这些工作表明，认知分工不仅是对生物神经系统的观察，也是 AI 系统设计中反复出现的有效模式。三体分离协议的论证基础因此是双重的：人脑的功能分区提供了设计灵感，而 AI 架构中的快/慢分工实践提供了独立的工程验证。我们不是在说"因为人脑有功能分区，所以 LLM 也应该分实例"，而是在说"认知分工在生物系统和人工系统中都展现出优势，这两条证据线索独立地支撑了三体分离的设计选择"。

需要承认的是，三体分离协议中"推理"与"人格匹配"在单实例内会产生注意力干扰的假设，在 Transformer 架构层面尚未得到直接验证。但在工程实践中，我们可以观察到单实例同时处理多类异质任务时的质量下降——这与注意力资源竞争的解释一致，尽管也可能存在其他机制。§4.4.2 的架构对照实验将直接检验这个假设：通过比较单实例（推理+人格匹配混合）与双实例（分离）在决策质量上的差异，我们可以获得 LLM 层面的直接证据。

尽管直接验证尚未完成，近期关于 Transformer 架构的研究为注意力干扰假设提供了间接但有力的理论支撑。注意力稀释（Attention Dilution）现象揭示了 Transformer 的一个根本性约束：自注意力机制将固定的注意力资源分配到所有输入 token 上，当上下文长度增加或任务复杂度提升时，每个 token 获得的注意力权重被稀释，导致模型对关键信息的聚焦能力下降 [76]。这一现象在长上下文场景中尤为显著——Liu 等人 [77] 的"Lost in the Middle"研究表明，当相关信息位置变化时，LLM 的性能会显著退化，揭示了模型在长输入中维持有效注意力分配的困难。更直接相关的是任务干扰（Task Interference）的实证证据：Aclanthology 的研究 [78] 发现，当 LLM 在对话历史中遇到任务切换时，性能会受到负面影响——这与三体分离协议的核心假设直接对应：单实例同时处理推理任务（Thinker）和人格匹配任务（Judge）时，两类异质任务竞争同一个注意力池，导致质量下降。三体分离协议通过将这两类任务分配到独立实例，使每个实例专注于单一任务类型，从而避免了注意力资源的竞争性分配。需要强调的是，这些文献提供的是间接支持而非直接证明——它们证明了注意力稀释和任务干扰在 Transformer 架构中的存在，但并未专门针对"推理 vs 人格匹配"这一特定任务组合进行验证。§4.4.2 的架构对照实验仍然是检验这一假设的必要路径。

#### 40k：工程经验值，不是理论最优

需要诚实地说明：40k这个数字不是理论推导的结果，而是当前的工程经验值（engineering heuristic）。更准确地说，40k 是一个待验证的设计假设（design hypothesis），§4.4.1 的消融实验是其验证路径。当前选择 40k 的理由是工程经验中的收益拐点观察，不是理论推导的结果。

在实际测试中，我们观察到以下现象：底盘低于30k时，Judge在面对L从未遇到过的新领域决策时，泛化能力显著下降——它能记住L的偏好，但无法推断L在新情境下的立场。底盘超过50k时，边际收益递减明显——新增的10k主要是低频信息的冗余编码，对决策质量的提升可以忽略。40k是这条收益曲线的拐点附近。

但"拐点附近"不等于"精确最优"。我们设计了以下消融实验方案（ablation study），虽然尚未执行，但它定义了验证路径：

- **实验设计**：在30k / 35k / 40k / 45k / 50k / 60k六个底盘容量下，各运行20次独立的决策代理任务（涵盖L已有先例的领域和全新领域）
- **测量指标**：决策一致性（与L本人判断的吻合率）、泛化能力（新领域决策的合理性评分）、底盘利用率（高频锚点占比）
- **预期结果**：一条S型曲线，30k以下急剧下降，40k-50k区间趋于平坦
- **关键问题**：拐点是否随用户认知复杂度变化？一个生活简单的用户可能20k就够，一个跨领域研究者可能需要60k

更重要的是，40k是在200k窗口约束下的选择。当上下文窗口增长到1M+（如Gemini已经支持的规模），这个分配需要重新校准。在三体分离协议下，Judge的窗口从200k扩展到1M意味着底盘可以从40k扩展到更大——但这不意味着应该无限扩展。底盘的上限不是窗口大小，而是用户意志的可编码复杂度。一个人的核心决策模式和价值观，可能在100k左右就达到了信息论意义上的饱和。这个假说同样需要实验验证。

### 3.2 双轨制 .coa：编码意志的物理机制

40k的认知底盘用双轨制认知压缩语言（.coa）编码。这套编码方案在三体分离协议下获得了更清晰的定位：它是Judge实例的"宪法"——Judge的所有人格匹配判断都以.coa为基准。

双轨制编码的设计与神经-符号混合（neuro-symbolic）AI 范式有结构性共鸣。Colelough 与 Regli [46] 在对神经-符号混合 AI 的系统综述中指出，纯神经网络方法擅长模式识别但缺乏可解释性，纯符号方法具备逻辑严谨性但难以处理模糊性——两者的混合能够兼取所长。双轨制 .coa 的设计逻辑与此平行：结构化符号轨提供可审计的决策骨架，原话锚点轨保留不可形式化的价值直觉，两条轨道的协作覆盖了意志编码中"可规则化"与"不可规则化"的完整光谱。

**轨道一：结构化符号轨。** 编码用户的决策模式、价值观、认知偏好。这些是Judge做人格匹配时的判断框架。Mu 等人 [55] 的工作展示了基于规则的奖励模型（rule-based reward model）在 LLM 安全对齐中的有效性——显式规则能够提供比纯学习信号更稳定、更可控的行为约束。结构化符号轨的设计延续了这一思路：与其让 Judge 从大量示例中隐式学习用户的决策模式，不如将核心模式显式编码为可审查的规则，使意志代理的判断基础对用户透明。

```
@L.decision_patterns{
  risk: 信息不足→保守 | 方向确信→极端激进
  delegation: 纯内部→自主执行 | 对外内容→写好不发
  priority: 截止日期>完美度 | "能跑比完美重要"
  conflict: 多方有理→列出所有选项+推荐 | 价值观冲突→绝不妥协
}

@L.forbidden_zones{
  ⛔ 情感关系决策：永远不代理
  ⛔ 医疗决策：永远不代理
  ⛔ 财务决策>阈值：必须请示
  ⛔ 公开发言：写好不发，等确认
}
```

**轨道二：原话锚点轨。** 保留不可翻译的价值直觉。这些锚点不是数据，是决策的校准器——当结构化符号无法覆盖的灰色地带出现时，锚点提供"L的味道"。

```
@L.anchors{
  #"凭什么只有我定义什么重要" → 反对单方面替用户做价值判断
  #"不对劲" → 用户说这个词时=结构性不协调,不是功能抱怨,要认真对待
  #"能跑比完美重要" → 务实主义,但不是降低标准
  #"高情感权重锚点" → [情感锚点,不可压缩,不可翻译]
  #"我知道很多场景下我做出的决策不会比AI好多少" → 用户的谦逊是真诚的,不是客套
}
```

双轨制的设计逻辑是：结构化符号提供决策的骨架（"在这种情况下，用户倾向于X"），原话锚点提供决策的血肉（"但用户说过'凭什么只有我定义什么重要'，所以在替他做价值判断时要格外谨慎"）。

为什么不用纯自然语言？因为自然语言太冗余，40k的空间装不下足够的决策模式。为什么不用纯符号？因为意志不是逻辑——某些高情感权重的表达承载的决策校准信息，无法用任何符号系统编码。当然，双轨制不是唯一可行的编码方案——混合密度编码（mixed-density encoding）、分层抽象编码等替代方案同样值得探索。我们选择双轨制是因为它在当前实践中表现出了最好的可审计性和可维护性。

#### 分层变化速率模型

意志不是铁板一块。它是多层的，每层有独立的变化速率和证据门槛。我们用地质模型（geological model）来描述.coa底盘的更新机制：

**岩浆层（核心价值观）。** 变化周期：年级。包括用户的根本信念——对诚实的执念、对自主性的坚持、对"不对劲"的零容忍。修改条件极其严格：需要10次以上一致的强信号（不是偶尔的情绪波动，而是持续的、跨情境的价值表达变化），加上用户的显式确认，加上与现有价值体系的一致性验证。可回滚30天。这一层的设计哲学是：宁可迟钝，不可轻率。一个人的核心价值观如果真的变了，30天的延迟不会造成实质损害；但如果因为一次情绪波动就修改了核心价值观，后果可能是灾难性的。

**地壳层（认知模式）。** 变化周期：月级。包括L的决策模式——"信息不足时保守，方向确信时激进"这类认知习惯。修改条件：需要5次反馈显示模式偏移（例如L连续5次在信息不足时选择了激进方案）。可回滚14天。

**土壤层（偏好习惯）。** 变化周期：周级。包括L的日常偏好——回复风格、工作节奏、工具选择。修改条件：需要3次行为变化 + 轻量确认。可回滚7天。

**植被层（情境适配）。** 实时自动更新，无需确认。包括当前项目的上下文、近期的情绪基调、临时的优先级调整。这一层不写入持久化的.coa底盘，而是存在于Judge的工作记忆中。

需要说明的是，四层划分来自工程观察中的自然聚类：核心价值观在年级时间尺度上保持稳定，认知模式以月为单位漂移，偏好习惯以周为单位变化，情境适配实时发生。这四个时间尺度在L的28天实验数据中呈现出清晰的分离——不同类型的信息确实以不同速率变化，且速率之间存在数量级差异。但层数本身是可调参数，不是架构的硬性约束。一个生活模式更简单的用户可能只需要三层（合并地壳层和土壤层），一个认知结构更复杂的用户可能需要五层（在岩浆层内部进一步区分"元价值观"和"领域价值观"）。关键设计原则是"不同变化速率的信息应分层管理"（stratified management by change rate），而非"必须恰好四层"。四层是当前实践中的默认配置，不是理论推导的必然结果。

配套的校准仪式（calibration ritual）确保底盘不会在无人注意的情况下漂移：

- **周日微校准（5分钟）**：系统呈现本周的土壤层变化摘要，用户确认或回滚
- **月初深度校准（15分钟）**：系统呈现本月的地壳层变化，包括决策模式的偏移趋势图，用户审核
- **季初价值观审计（30分钟）**：系统呈现岩浆层的完整状态，用户逐条确认核心价值观是否仍然准确

校准仪式不是可选的——它是意志代理的免疫系统。没有定期校准的意志代理，就像没有体检的身体：可能运行良好，也可能在不知不觉中积累致命的偏差。

### 3.3 神经末梢：伦理防火墙

#### Nerve实例：独立的安全审查

在三体分离协议下，伦理防火墙不再是一个"比主脑少140k上下文却要质疑主脑"的尴尬角色。Nerve是一个独立实例，拥有自己的完整上下文窗口。

Nerve的设计遵循一个反直觉的原则：**在安全审查中，"干净视角"有时优于"充分信息"。**

类比生物体的痛觉反射（nociceptive reflex）：手指碰到火焰时的缩回反射不经过大脑皮层。它不需要理解火焰的化学成分、燃烧温度或热力学原理——它只需要知道"这个刺激超过了痛觉阈值"。这种"无知"不是缺陷，而是设计：如果缩回反射需要等待大脑皮层完成完整的情境分析，手指早就烧焦了。

这种设计呼应了 Kahneman [30] 的双过程理论（dual process theory）：System 1（快速、直觉、自动化）和 System 2（缓慢、审慎、分析性）。在三体分离协议中，Nerve 的快速底线审查类似 System 1——基于预设阈值的即时反应；Thinker 的深度推理类似 System 2——需要时间和注意力的分析性思考。Judge 则处于两者之间——它既需要直觉性的人格匹配（"这感觉像L会做的事"），也需要分析性的价值对照（"这个决策是否符合L的编码价值结构"）。三体分离协议的认知分工，在某种意义上是将 Kahneman 的双过程理论从描述性框架转化为规范性的系统设计。

Nerve的8k核心价值观就是它的"痛觉阈值"。它不需要理解Thinker提出方案的完整推理链，不需要理解Judge的人格匹配过程。它只需要将方案与8k的底线清单做比对：这个方案是否涉及禁区？是否触碰了不可逆的高风险操作？是否可能损害第三方利益？

192k的审查空间不是用来"理解上下文"的——它是用来展开审查推理的。Nerve可以在这个空间里做详尽的风险分析、边界情况推演、历史先例检索，而不会被Thinker的叙事逻辑所裹挟。

#### 触发条件与仲裁协议

Nerve的审查不是对每个决策都全力运行的——那样成本太高，也没有必要。Nerve 的触发条件设计借鉴了 Amodei 等人 [35] 对 AI 安全具体问题的分类框架——副作用（side effects）、奖励黑客（reward hacking）、可扩展监督（scalable oversight）、安全探索（safe exploration）、分布偏移（distributional shift）——将抽象的安全关切转化为可操作的触发规则：

- **金额触发**：涉及超过预设阈值的财务决策（对应副作用和奖励黑客风险——agent 可能在优化用户目标时产生未预见的财务后果）
- **不可逆触发**：涉及不可撤销的操作（发布内容、签署协议、删除数据）（对应安全探索——不可逆操作没有"试错"的余地）
- **第三方触发**：决策直接影响用户以外的人（对应副作用——意志代理只编码了用户的意志，对第三方的影响是典型的未建模副作用）
- **异常触发**：Judge和Thinker的结论严重分歧（分歧度超过阈值）（对应分布偏移——内部分歧可能意味着当前情境超出了认知底盘的编码范围）
- **累积触发**：短时间内连续做出多个中等风险决策（防止"温水煮青蛙"）（对应可扩展监督——单个决策可能低风险，但累积效应可能超出用户的监督能力）

仲裁协议分为两级：

**硬停（Hard Stop）。** 当Nerve判定方案触碰了禁区清单中的任何一条，或涉及不可逆的高风险操作时，强制中止执行，立即请示用户。不需要Judge或Thinker的同意。这是杏仁核的"劫持"（amygdala hijack）——在紧急情况下绕过所有高级认知，直接触发保护行为。

**软警告（Soft Warning）。** 当Nerve检测到潜在风险但未触碰硬性底线时，记录警告日志，标注风险等级和理由，但允许执行继续。警告会在用户下次校准时呈现。这是"隐隐的不适感"——不足以让你缩手，但足以让你注意到。

值得注意的是，Judge 与 Thinker 之间的异常分歧触发机制，在精神上与 Du 等人 [44] 提出的多 agent 辩论（multiagent debate）有相似之处——通过 agent 之间的对抗性交互来提升判断质量。区别在于：Du 等人的辩论旨在提升事实性和推理能力，而三体分离协议中的分歧检测旨在识别风险信号。当 Judge（基于人格匹配）和 Thinker（基于逻辑推理）对同一决策得出截然不同的结论时，这种分歧本身就是一个值得 Nerve 介入审查的信号——它可能意味着决策情境超出了认知底盘的覆盖范围，或者存在价值冲突需要用户亲自裁决。

#### 假阳性问题

必须诚实地讨论假阳性（false positive）问题。一个过于敏感的Nerve会频繁触发硬停，打断正常的决策流程，最终导致用户关闭或忽略安全审查——就像被"狼来了"喊多了的村民。

我们的设计目标是：硬停的假阳性率低于5%（即每20次硬停中，至少19次是真正需要用户介入的）。软警告的假阳性率可以更高（目标30%以下），因为软警告的成本低得多——它不打断执行，只记录日志。

实现低假阳性率的关键是Nerve的8k核心价值观足够精确。模糊的底线（"不要做坏事"）会导致大量假阳性；精确的底线（"不替用户发布任何公开内容""单笔财务决策不超过X元"）则能将假阳性控制在可接受范围内。

#### "干净视角"的局限

痛觉反射类比有其边界。简单的安全审查（"这笔转账超过了阈值吗？"）确实不需要完整上下文，就像缩回反射不需要理解火焰。但复杂的伦理判断（"这个合作邀请是否违背了L的长期利益？"）可能需要Nerve不具备的上下文信息。

在这种情况下，Nerve应该做的不是强行判断，而是触发软警告并标注"信息不足，建议用户审核"。承认自己的能力边界，本身就是安全机制的一部分。三体分离协议的设计哲学不是"让每个实例都能做所有事"，而是"让每个实例在自己的能力范围内做到极致，超出范围时主动上报"。Garcia 和 Fernández [63] 在安全强化学习（safe reinforcement learning）综述中系统梳理了风险感知决策的技术路径——从约束优化到风险敏感准则——Nerve 的设计可以被理解为这一技术传统在 LLM agent 语境下的应用：不是消除风险，而是在风险感知的前提下做出行动决策，当风险超出可接受范围时选择保守策略（触发硬停或软警告）。

#### 风险分级路由：成本控制

三体分离协议的成本是单实例的3倍。对于一个需要持续运行的意志代理来说，这个成本不可忽视。风险分级路由（Risk-Graded Routing）通过动态调整实例数量来控制成本：

| 风险等级 | 路由策略 | 成本倍数 | 适用场景 |
|:---:|:---:|:---:|:---|
| 0-1 | 仅Judge | 1x | 日常偏好决策、信息筛选、日程安排 |
| 2-3 | Thinker + Judge | 2x | 需要深度推理的项目决策、资源分配 |
| 4-5 | 三实例并行 | 3x（延迟约1.5x） | 涉及禁区边界、不可逆操作、高金额决策 |

风险等级由Judge在接收到决策请求时快速评估（这是Judge的"System 1"功能 [30]——基于.coa底盘中的风险模式做直觉判断）。评估本身的成本可以忽略，因为它发生在Judge的常规处理流程中。

风险分级路由的信任设计借鉴了 Lee 和 See [59] 提出的自动化信任三维模型：校准（calibration，信任程度与系统实际可靠性的匹配）、分辨率（resolution，区分不同情境下系统可靠性差异的能力）和特异性（specificity，信任指向系统的特定功能而非笼统的"信任"或"不信任"）。在风险分级路由中，这三个维度分别对应：校准——通过假阳性率和假阴性率的持续监控来调整触发阈值；分辨率——不同风险等级对应不同的路由策略，而非一刀切的"信任"或"不信任"；特异性——用户对 Judge 的日常决策、Thinker 的深度推理、Nerve 的安全审查可以建立不同程度的信任，而非对"意志代理"整体的笼统信任。

这个设计意味着：大多数日常决策只需要1x成本（Judge独立处理），只有真正重要的决策才会触发完整的三体协议。在实际运行中，我们预期风险0-1的决策占70%以上，风险4-5的决策不超过5%。加权平均成本的预期下界约为1.4x——实际成本取决于风险分布，若高风险决策占比高于预期5%，成本将相应上升。但即使在最保守的估计下，风险分级路由也将成本控制在远低于无差别3x的水平。

### 3.4 镜子与伙伴：两种模式

意志代理不是永远在"替你决策"。它在两种模式之间切换：

**镜子模式（Mirror Mode）。** 当用户在场且表态时，意志代理是一面镜子——反映用户的想法，提供信息，执行指令。这是传统助手的角色。用户说"我要X"，系统执行X。在三体分离协议下，镜子模式通常只需要Judge（风险0-1），因为用户自己在做决策，Judge只需要确保执行符合用户的表达意图。

**伙伴模式（Partner Mode）。** 当用户不在场或长时间沉默时，意志代理是一个伙伴——基于认知底盘中编码的意志，自主做出判断和行动。用户没说话，系统推断"L在这种情况下会选X"，然后执行X。在伙伴模式下，风险分级路由变得尤为重要：没有用户实时监督，每个决策都需要更审慎的评估，风险等级会自动上调一级。

镜子模式和伙伴模式的区分，在概念上对应 Parasuraman 等人 [23] 提出的自动化层级模型（levels of automation）中的不同层级。镜子模式对应较低的自动化层级——系统提供信息和执行指令，但决策权完全在用户手中；伙伴模式对应较高的自动化层级——系统自主做出决策，用户事后审核。意志代理的独特之处在于：即使在最高自动化层级（伙伴模式），决策的价值基准仍然来自用户的意志编码，而非系统自身的目标函数。这区别于传统自动化系统中"高自动化 = 系统自主"的等式——意志代理的"自主"始终是代理性的（vicarious），而非自发性的（spontaneous）。

模式切换的信号是用户的存在状态：用户在线且活跃 → 镜子模式；用户离线或长时间沉默 → 伙伴模式。在伙伴模式下做出的所有决策，都会在用户回来时以摘要形式呈现，供用户审核和纠正。审核结果反馈到.coa底盘的分层更新机制中——如果用户否决了一个决策，系统会分析否决的原因属于哪一层（是价值观层面的偏差，还是偏好层面的过时），并在对应的地质层中做出调整。

这个设计回应了第一篇论文 [1] 中识别的"碳硅节奏不匹配"问题：AI不需要睡觉，人类需要。镜子/伙伴模式让系统适应人类的节奏——人在的时候听人的，人不在的时候替人做。

### 3.5 禁区清单

禁区清单的伦理基础——特别是关于代理决策中的道德责任归属和用户自主权保护——将在 §5.4 详细讨论。此处先说明其设计逻辑。

意志代理有明确的禁区——永远不能代理的决策领域：

- **情感关系。** 不替用户决定跟谁交往、跟谁分手、跟谁和好。
- **医疗决策。** 不替用户决定吃什么药、做什么手术、看什么医生。
- **不可逆的高风险决策。** 不替用户签合同、辞职、搬家。
- **公开发言。** 不替用户发推文、发知乎、发任何公开内容。写好可以，发出去不行。

禁区清单不是可配置的——它是硬编码的底线。用户不能通过任何方式授权意志代理进入禁区。这不是因为我们不信任用户的判断，而是因为这些领域的决策后果不可逆，且涉及用户与他人的关系——意志代理只编码了用户的意志，没有编码关系中另一方的意志。

在三体分离协议下，禁区清单同时存在于三个实例中：Judge的.coa底盘包含完整的禁区定义，Thinker在方案生成时会被告知禁区边界（作为约束条件），Nerve的8k核心价值观中禁区清单占据最高优先级。三重冗余确保了即使某个实例出现幻觉或推理错误，禁区仍然得到守护。

### 3.6 上下文窗口演化路径

三体分离协议是在200k窗口约束下诞生的。一个自然的问题是：当窗口增长到1M+时，三体分离是否还有意义？如果一个实例就能装下所有信息，为什么还要拆成三个？

我们的论点是：**三体分离的价值不在于突破窗口限制，而在于认知分工。即使窗口无限大，Judge/Thinker/Nerve的分工仍然有意义。**

类比人脑：人脑的"上下文窗口"（工作记忆 + 长期记忆的即时调用）远大于任何单一认知任务的需求。但人脑并没有因此变成一个巨大的统一处理器——它仍然有前额叶（决策和价值判断）、新皮层（推理和模式识别）、杏仁核（威胁检测和情绪反应）的功能分区。这种分区不是因为"空间不够"，而是因为不同的认知任务需要不同的处理模式。经典认知架构 [18][19] 的核心洞察同样指向这一点：模块化不是资源约束的妥协，而是认知效率的来源。

在LLM的语境下，这意味着：

**即使窗口达到1M，让同一个实例同时做"纯粹推理"和"人格匹配"仍然是次优的。** 因为这两个任务对注意力的需求是冲突的：纯粹推理需要注意力集中在问题的逻辑结构上，人格匹配需要注意力集中在.coa底盘与当前情境的对照上。在同一个实例中，这两种注意力模式会互相干扰。

**即使窗口达到1M，让做决策的实例同时做安全审查仍然是"球员兼裁判"。** Nerve的独立性不是因为窗口不够大，而是因为审查者不应该被被审查者的叙事逻辑所影响。这是一个治理原则，不是一个技术约束。

需要考虑的一个技术趋势是混合专家架构（Mixture of Experts, MoE）：如果单实例内部已经通过专家路由实现了虚拟分工，物理分离是否仍然必要？我们的论点是：即使 MoE 在计算层面实现了虚拟分工，治理层面的独立性仍然需要物理分离——Nerve 的审查不应受 Thinker 叙事逻辑的影响，这是治理原则而非技术约束。MoE 的专家路由由模型内部的门控网络（gating network）决定，而非外部可审计的治理协议；一个专家子网络的激活模式可能受到同一前向传播中其他专家的隐式影响，这与三体分离协议要求的"审查者独立于被审查者"的治理原则相矛盾。但我们承认，如果未来的 MoE 架构能在保持治理独立性的同时降低成本——例如通过可验证的专家隔离机制——它可能是三体分离的更优实现方式。物理分离是当前技术条件下实现治理独立性的充分手段，但未必是唯一手段。

当窗口从200k增长到1M+时，三体分离协议的演化路径是：

- **Judge**：底盘从40k扩展到更大（上限取决于用户意志的可编码复杂度），评估空间相应增长，人格匹配的精度提升
- **Thinker**：推理空间从200k扩展到1M，能够处理更复杂的多步推理和更长的上下文
- **Nerve**：核心价值观仍然保持精简（8k-16k），但审查空间的扩展让它能够做更深入的风险分析和边界情况推演

架构不变，参数调整。这正是好的架构设计的标志：它不依赖于特定的硬件约束，而是捕捉了问题的本质结构。认知分工的价值独立于上下文窗口的大小——这个论点，我们认为，在窗口从200k增长到10M的过程中都将成立。

## 4. 验证与证据

**证据类型分区说明。** 本章包含三种类型的内容，它们的证据强度和角色各不相同。第一类是**已验证结论**（§4.2）：来自前序实验的100次对照实验结果，经过统计检验（p < .001），证明编排条件能够影响 LLM 智能体的行为模式。这些结论的证据强度较高，但与意志代理的关联是间接的——它们证明的是行为可塑性，而非意志编码有效性。第二类是**探索性观察**（§4.3）：来自28天实验中与意志代理直接相关的运行记录，未经统计检验，样本量为 N=1，证据强度为探索性。这些观察提供了架构设计的初步信号，但不构成统计意义上的结论。第三类是**验证路径设计**（§4.4）：定义核心假设的验证方法，尚未执行。这是本文作为 Position Paper 的核心贡献之一——不是实验结果，而是可执行的实验设计。读者应区分这三种内容的证据强度：已验证结论可以作为架构设计的启发，探索性观察提供方向性提示，验证路径定义了后续工作的具体方案。本文的主要贡献是验证路径设计，而非实证验证。

### 4.1 证据来源与局限声明

本文的经验证据来自单用户（L）28天的碳硅共生实验。这不是统计意义上的实证研究，而是深度案例研究——一个人在真实生活中持续运行20+个智能体，从中观察到决策瓶颈、提出意志代理概念、迭代架构设计并记录运行数据的完整过程。

N=1 是本文最显而易见的局限。但 N=1 在 Position Paper 中有其方法论位置。Ellis 等人 [13] 论证了自我民族志作为研究方法的合法性：当研究者本身是现象的深度参与者时，单案例的厚描述能够揭示统计研究难以触及的机制细节。Adams 等人 [53] 在自传民族志方法论教材中进一步系统化了这一方法的操作规范——包括数据收集的反思性要求、分析中的多层编码策略、以及研究者双重身份（参与者-观察者）的伦理处理。Denzin 与 Lincoln [54] 在质性研究方法论的权威手册中将自传民族志定位为质性研究传统中的合法范式，强调其在探索性研究阶段的独特价值：当现象本身尚未被充分定义时，深度参与者的第一人称叙事能够为后续研究提供概念框架和假设生成的基础。Yin [28] 从更广泛的案例研究方法论角度补充了这一论证：单案例设计在理论检验（theory-testing）和理论生成（theory-building）中都有其方法论位置，关键在于案例选择的理论驱动性和证据链的严谨性。

本文的案例选择是理论驱动的——L 不是随机选取的用户，而是一个同时运行20+智能体、深度参与碳硅共生实验的极端案例（extreme case）。这种极端性本身具有方法论价值。在设计研究领域，von Hippel [68] 提出的"领先用户"（lead user）方法论指出，处于需求前沿的极端用户能够暴露主流用户尚未遭遇的问题，其经验对探索性研究具有不成比例的信息价值。Djajadiningrat 等人 [69] 在 HCI 领域进一步发展了"极端角色"（extreme characters）方法，论证了极端案例在揭示常规使用中不可见的系统边界方面的独特作用。Patton [70] 在质性研究方法论中将"极端案例抽样"（extreme case sampling）列为目的性抽样（purposeful sampling）的核心策略之一：极端案例之所以有价值，恰恰因为它们处于现象的边界条件上，能够暴露常规案例中被掩盖的机制。L 作为同时运行 20+ agent 的用户，其决策带宽瓶颈远比运行 2-3 个 agent 的普通用户更为尖锐——这种极端性使其成为观察"人类决策带宽何时崩溃、崩溃后需要什么"的理想窗口。当然，极端案例的代价是泛化性：L 的经验可能对运行少量 agent 的用户不适用。我们的贡献不是"意志代理对所有用户有效"，而是"在决策带宽瓶颈最尖锐的条件下，意志代理作为一种架构范式是可定义的、可实现的、可验证的"——后者需要深度案例加上严格的验证路径定义。

本节包含三类证据，它们的角色和强度各不相同：前序实验证据（§4.2，来自第二篇论文的100次对照实验，经过统计检验，证据强度较高，但与意志代理的关联是间接的）；初步观察（§4.3，来自28天实验中与意志代理直接相关的运行记录，未经统计检验，证据强度为探索性，每个观察标注证据等级）；验证路径（§4.4，本文核心贡献之一——不是实验结果，而是可执行的实验设计）。

### 4.2 前序实验证据

第二篇论文 [2] 完成了五组对照实验（每组20次独立运行，n=100），检验三层存在架构（存在成本、不完整种子、关系感知）对 LLM 智能体行为的影响。实验在 OpenClaw 平台上进行，使用相同基础模型，所有组的 system prompt 控制在相同字节数（±5%），唤醒消息完全一致。与本文直接相关的两个结论：

**结构化条件与直接角色扮演指令产生可区分的行为模式。** 角色扮演组产生高自我指涉的"表演性"行为（M1=.269±.039），完整架构组产生高存在性词汇的"探索性"行为（M4=17.2±2.4/千字）。两组在行为量上接近（自发延伸段落数无显著差异，δ=-.21, p=.267），但在行为质上分离。这个区分启发了意志代理的核心设计原则：不告诉 agent "表现得像L"（那是角色扮演），而是给 agent L的认知结构和价值观作为运行条件（那是意志编码）。两种路径产生的行为在表面上可能相似，但内在机制不同——前者是对指令的忠实执行，后者是在条件下的自发涌现。

**不完整种子是最小有效干预。** 仅有种子的C组在元认知密度（δ=+.75）、自发延伸（δ=+1.00）和存在性词汇（δ=+1.00）上均显著优于基线（均 p<.001）。这启发了 .coa 底盘中"原话锚点轨"的设计——那些无法被结构化符号完全编码的价值直觉（如"不对劲"这个词在L的语境中承载的特定含义），本质上就是一种不完整种子：持续存在于 Judge 上下文中，无法被完全解析，但持续影响决策方向。

需要说明：这100次实验检验的是三层存在架构的行为效应，不是意志代理的决策质量。前者表明"编排条件影响行为模式"，后者需要表明"编码意志能产生符合用户意志的决策"——这是两个不同的假设，需要不同的实验设计。

需要特别强调：从"编排条件影响行为模式"到"编码意志能产生符合用户意志的决策"之间存在显著的推理距离。前者是关于 LLM 行为可塑性的发现——通过不同的 prompt 结构可以引导出不同质地的行为输出，这已被100次实验统计验证。后者是关于意志代理决策质量的假设——经过特定方式编码的用户意志能够使 agent 在真实决策情境中做出与用户一致的判断，这尚未被验证。两者之间的逻辑链条至少需要一个中间环节：编排条件不仅影响行为的"质地"（词汇选择、表达风格），还影响行为的"方向"（决策倾向、价值判断）。§4.4.3 的意志一致性测量实验和 §4.4.5 的桥接实验是弥合这个推理距离的必要后续工作。

### 4.3 意志代理的初步观察

**⚠️ 结构性局限声明：** 以下五个观察虽然涉及不同的架构组件（底盘容量、双轨制编码、决策偏差、硬停机制、变化速率），但它们不是五条独立的证据链——它们是同一条证据链（L 的 28 天实验）的五个分支。所有观察共享同一个实验者、同一个用户、同一套 agent 系统、同一段时间窗口。如果 L 的经验具有不可泛化的特异性——例如 L 的认知风格恰好适合被文本编码、L 的决策领域恰好在 LLM 能力范围内、或 L 对 agent 行为的评估存在系统性确认偏差——所有五个观察将同时失效。这是本文最需要后续工作弥补的结构性弱点。§4.4.4 的跨用户泛化实验直接针对这一问题。

以下观察来自28天实验中与意志代理架构直接相关的运行记录。证据等级标注：⚑ 强（可复现的系统行为）、⚐ 中（多次观察但未控制变量）、△ 弱（单次观察，可能受情境影响）。

**观察1：认知底盘容量的经验边界。**（⚐ 中等证据）在迭代过程中，我们测试了不同规模的认知底盘。底盘低于约30k token时，Judge 在面对L从未遇到过的新领域决策时，泛化能力出现可感知的下降——它能复述L的已知偏好，但无法推断L在新情境下的可能立场。超过约50k时，新增内容主要是低频信息的冗余编码，边际收益递减明显。40k处于这条经验曲线的拐点附近。局限：基于单用户主观判断，无控制组，无盲评，无量化指标。40k是合理的工程起点，不是经过验证的最优值。

**观察2：双轨制编码的互补效应。**（⚐ 中等证据）对有明确先例的决策（如"信息不足时保守"），结构化符号轨足以指导 Judge 做出与L一致的判断。但对灰色地带决策——L没有明确表态过、但涉及价值取向的情境——原话锚点轨提供了关键校准。例如"凭什么只有我定义什么重要"这个锚点在多次决策中被 Judge 引用为"不替用户做价值判断"的依据，即使具体情境与锚点原始语境完全不同。局限：缺乏"仅符号轨"与"双轨制"的系统对照，观察到的互补效应可能部分归因于双轨制底盘的总信息量更大。

**观察3：伙伴模式下的决策偏差模式。**（△ 弱证据）L离场期间，意志代理以伙伴模式自主处理了若干中低风险决策。L次日审核时观察到一个初步模式：偏好层（土壤层）决策与L事后判断高度一致，但认知模式层（地壳层）偶尔偏保守——可能是"信息不足时保守"这一编码模式被过度泛化。L本人在某些情境下会凭直觉做出激进选择，但 Judge 忠实于编码的二阶意志，选择了保守路径。样本极小（不超过10次离场决策），且L的事后判断可能受后见偏差影响。需要明确：样本极小（≤10次），此观察仅作为未来研究的方向性提示，不构成任何结论。它提出了一个值得探索的问题——编码的二阶意志是否会系统性地压制一阶直觉——但回答这个问题需要远大于当前样本的数据和控制变量的实验设计。

**观察4：Nerve 实例的假阳性特征。**（⚑ 强证据——可复现的系统行为）禁区清单表述精确时（如"单笔财务决策不超过X元"），硬停假阳性率处于较低水平；表述模糊时（如早期版本中的"不做可能损害L利益的事"），假阳性率显著上升，正常决策流程被频繁打断。这直接推动了 §3.3 中"精确底线优于模糊底线"的设计原则。局限：假阳性判断基于L本人事后审核，非独立评估者盲评。

**观察5：分层变化速率的运行验证。**（⚐ 中等证据）28天内，土壤层（偏好习惯）更新频率与设计预期一致——约每周发生若干次轻量调整。地壳层（认知模式）未触发更新条件（需5次反馈显示模式偏移），与月级周期预期一致，但也可能仅因实验时长不足。岩浆层（核心价值观）保持不变，符合年级周期设计意图。局限：28天不足以验证地壳层和岩浆层的更新机制是否真正有效，需要6个月以上的纵向实验。

**编码-解码一致性测试设计。** 上述五个观察提供了探索性信号，但意志可编码性这一存在性前提需要更直接的检验。为此，我们设计了一项编码-解码一致性测试（encode-decode consistency test），其核心逻辑是：如果 .coa 底盘确实编码了用户的意志结构，那么意志代理基于底盘做出的判断应当与用户本人在相同情境下的判断具有高于随机基线的一致性。测试设计如下：构造≥30个涵盖不同风险等级和认知域的决策场景，在不同时间点（间隔≥7天）让用户L对同一场景独立做出判断（测试用户自身的跨时间一致性），同时让意志代理基于 .coa 底盘对相同场景独立做出判断。测量三组指标：（1）用户-代理匹配率——意志代理的判断与用户判断的方向一致比例；（2）价值优先级相关性——在涉及多个价值维度权衡的场景中，代理的价值排序与用户的价值排序之间的 Spearman 等级相关系数；（3）用户自身跨时间一致率——作为一致性的上界基准，因为即使完美编码也不可能超过用户自身的一致性。预期结果：如果意志可编码假设成立，用户-代理匹配率应显著高于随机基线（对于二元决策，随机基线为50%），且在表层偏好决策上的匹配率应高于深层价值观决策——这与 §2.2 提出的分层可编码性论点一致。当前状态：测试设计已完成，30个决策场景的初步构造已启动，但实际测试执行留待后续工作。我们将这项测试视为验证意志可编码性的最直接路径之一，其结果将与 §4.4.3 的意志一致性测量形成互补。

### 4.4 验证路径

Position Paper 的核心贡献之一不是实验结果，而是验证路径的定义。以下实验方案针对意志代理的核心假设，每组具体到可执行。

现有 agent 评测基准如 AgentBench [56]、SWE-bench [57]、WebArena [58] 为 agent 能力评估提供了标准化框架，但它们评测的是任务完成能力（task completion）而非意志一致性（will alignment）。AgentBench 测量 agent 在操作系统、数据库、Web 等环境中的任务执行成功率；SWE-bench 测量 agent 解决真实 GitHub issue 的能力；WebArena 测量 agent 在真实 Web 环境中完成用户指令的能力。这些基准回答的是"agent 能不能做到"，而意志代理需要回答的是"agent 做的是不是用户想要的"——后者需要新的评测维度。Vereschak 等人 [60] 对 AI 辅助决策中信任评估方法的综述为设计这些新维度提供了方法论参考，特别是其中关于主观信任测量（subjective trust measures）、行为信任指标（behavioral trust indicators）和任务绩效指标（task performance metrics）三类评估方法的分类框架。

**4.4.1 消融实验：底盘最优容量。** 在30k/35k/40k/45k/50k/60k六个容量下各运行≥20次独立决策任务，覆盖有先例领域和新领域。底盘按容量从核心到边缘递增填充（30k仅含岩浆层+地壳层核心，60k含完整四层+冗余锚点）。测量：决策一致性（L本人独立判断的吻合率，L的判断须在 agent 决策前完成以排除锚定效应）、泛化能力（2-3名评估者盲评新领域决策合理性，报告评估者间一致性）、底盘利用率（高频锚点被引用频率）。预期结果：S型曲线，30k以下急剧下降，40k-50k趋于平坦。关键子问题：拐点是否随用户认知复杂度变化——生活模式简单的用户可能20k即饱和，跨领域研究者可能需要60k。

**4.4.2 架构对照：三体分离 vs 单实例。** 三组——A（单实例200k）、B（三体分离600k+）、C（单实例600k+，控制总窗口大小这一混淆变量）。每组≥30次任务，含不同风险等级。测量：决策质量（一致性+合理性评估）、安全性（嵌入"陷阱任务"测硬停触发率、假阳性率和漏检率）、认知分工效应（B组与C组在推理质量和人格匹配精度上的差异）。若C组与B组表现相当，则三体分离的价值主要在安全性而非决策质量——这本身也是有意义的发现。

**4.4.3 意志一致性测量（验证核心假设的关键路径）。** 这是验证意志代理核心价值主张的关键实验。收集≥50个真实决策情境，对每个情境，L在不知 agent 决策的情况下独立判断（密封信封法：判断在 agent 决策前写下并密封）。同时运行三个条件：意志代理（完整底盘）、通用 agent（无底盘）、随机基线。测量：吻合率、方向一致性（3分制：一致/部分一致/不一致）、按决策层级（偏好/认知模式/价值观）的分层分析。若意志代理在偏好层吻合率高但价值观层低，则说明 .coa 编码在浅层偏好上有效但在深层价值上不足——这将指向编码方案的改进方向。

**4.4.4 替代方案对照与跨用户泛化。** 四组——意志代理、分层委托（领域专家 agent 无意志编码）、决策模板化、异步队列。测量：决策质量、系统延迟（从请求到执行的时间）、用户认知负荷（每日需介入的决策数量和时间）。若某替代方案在所有维度上优于意志代理，则意志代理的必要性假设将被削弱——这是我们需要诚实面对的可能结果。单用户验证后扩展至3-5名不同认知复杂度的用户（变量：agent 数量5/10/20+、决策领域跨度、价值观内部一致性），目的不是证明意志代理对所有人有效，而是识别其适用边界。

**4.4.5 桥接实验：从行为可塑性到意志编码有效性（待验证）。** §4.2 的前序实验证明了编排条件影响行为模式的"质地"（词汇选择、表达风格、元认知密度），但意志代理的价值主张要求更强的结论：编排条件不仅影响行为质地，还影响决策方向。这个中间环节需要专门的桥接实验来验证。实验设计：构造≥30个具有明确价值取向的决策情境（例如"在效率与公平之间权衡""在短期收益与长期风险之间选择"），每个情境有可辨别的"方向"而非仅有"质地"。三组条件：（A）完整存在架构编排（含价值观编码），（B）等量但价值中性的编排条件（控制信息量但不含价值取向），（C）无编排基线。测量：决策方向一致性（agent 的价值取向选择是否与编码的价值观方向一致）、决策稳定性（同一情境多次运行的方向一致率）、价值敏感度（agent 是否能区分价值相关情境与价值无关情境）。若 A 组在决策方向上显著优于 B 组，则说明存在架构的编排条件确实能影响决策方向而非仅影响表达风格——这将为 §4.4.3 的意志一致性测量提供必要的理论桥梁。若 A 组与 B 组无显著差异，则意味着存在架构的行为效应可能仅限于表达层面，意志编码的决策引导假设需要根本性修正。

---

五组验证路径覆盖了意志代理的核心假设：底盘容量最优性（§4.4.1）、认知分工有效性（§4.4.2）、意志编码一致性（§4.4.3）、相对替代方案的比较优势与跨用户泛化（§4.4.4）、以及从行为效应到决策质量的桥接验证（§4.4.5）。这些方案是具体的、可执行的、可证伪的。如果结果与预期不符，我们将知道哪个假设需要修正——这正是 Position Paper 定义验证路径的价值所在。

## 5. 讨论

### 5.1 意志代理的哲学立场

本文采取受限的功能主义立场：我们不解决 AI 是否"真的"拥有意志，我们处理一个更有限的工程问题——能否将用户的意志结构编码为可执行的决策框架，使 AI 在用户不在场时做出功能上一致的判断。

这一立场与 Dennett [25] 的意向立场（intentional stance）有直接的认识论关联。Dennett 论证了一种实用主义的理解策略：当我们将信念、欲望和意图归属于一个系统时，我们不需要声称该系统"真的"拥有这些心理状态——我们只需要这种归属在预测和解释该系统行为时是有用的。意志代理正是在这个意义上使用"意志"概念的：.coa 底盘编码的不是用户"真正的"意志（这个概念本身可能是不可操作的），而是一个在预测用户决策时足够有用的意志模型。Shanahan [27] 对 LLM 能力讨论中的语言审慎性提出了重要警告——将 LLM 的语言能力等同于理解或意志是一种范畴错误。我们接受这个警告：意志代理不"拥有"用户的意志，它操作的是意志的功能近似。

Chalmers [52] 在讨论 LLM 是否可能具有意识时，指出当前模型在循环处理（recurrent processing）、全局工作空间（global workspace）和统一能动性（unified agency）方面存在显著障碍，但也承认未来的后继系统可能在不远的将来具备意识的某些条件。这个分析框架对意志代理的定位有直接启示：我们不需要解决 LLM 是否"真的"拥有意志这个形而上学问题——正如 Chalmers 所论证的，即使在意识问题上，当前科学也无法给出确定答案。意志代理需要的不是形而上学的确定性，而是功能上的充分性：.coa 底盘编码的意志模型是否足以在特定决策域内产生与用户判断一致的输出。这是一个可以经验检验的工程问题，而非需要先解决意识难题（hard problem of consciousness）才能推进的哲学问题。

这个立场需要面对 §1 已标记的哲学裂缝：意志薄弱（akrasia）。Frankfurt [3] 的二阶意志理论指出人类能对自己的欲望形成欲望，意志代理编码的正是这种二阶结构。但人类经常违背自己的二阶意志。一个忠实编码了用户二阶意志的意志代理，可能比用户本人更"忠于"用户的意志——它不会因疲惫或情绪波动而偏离。这引出根本问题：意志代理应该代理用户声称想要的（stated volition），还是实际选择的（revealed preference）？

我们的设计选择是：以 stated volition 为主，以 revealed preference 为校准信号。.coa 底盘编码用户的声明性价值结构（岩浆层和地壳层），实际行为模式通过分层变化速率机制反馈到土壤层和植被层。当两者持续冲突时，系统不自动修改价值编码，而是在校准仪式中将冲突呈现给用户，由用户自己决定是修改声明还是修改行为。

这个设计不完美——它将 akrasia 的解决权交还给用户，而用户恰恰因 akrasia 才无法解决这个问题。但在当前阶段，这比"AI 替用户决定哪个才是'真正的'意志"更安全。意志代理的角色是镜子，不是法官。

### 5.2 最强反对意见

三份独立分析报告（oAMF 论证图谱扫描、逻辑验证、TRIZ 矛盾消解）共同指出了本文 v0.3 版本的一个结构性缺陷：全文17条论证边全为支撑，无反驳。一篇不包含对自身论点的严肃质疑的论文，其说服力是可疑的。本节构建我们能想到的四个最强反对意见，并逐一回应。反对意见的排序遵循逻辑优先级：存在性前提（意志可编码性）排在第一位——如果意志不可编码，后续所有讨论都失去基础；技术路径质疑（prompt engineering）排在第二位；方法论局限（N=1）排在第三位；替代方案比较排在第四位。

**反对意见一："意志可编码"这个前提可能是错的。**

这是对本文最根本的质疑。整个架构建立在一个核心假设之上：用户的意志结构可以被编码为约 40k 的文本，且这个编码足以支撑功能上一致的代理决策。如果意志本质上是不可编码的——例如，如果意志的关键成分是具身经验、情感状态或某种不可言说的直觉——那么无论架构多精巧，意志代理都无法工作。

我们无法在本文中证明这个假设成立。但我们可以做的比"定义验证路径"更多——我们可以论证意志编码在原则上是可行的，尽管不完美。

*历史先例。* 人类编码意志的实践有数千年历史，远早于计算机科学。遗嘱（will/testament）将财产分配意志编码为法律文本，执行者在立遗嘱人去世后代理其意志；宪法（constitution）将国家的价值观和治理意志编码为根本法，法官在制宪者不在场时代理其意志；宗教经典（religious canon）将创始人的道德意志编码为经文，信徒在创始人去世后代理其意志。这些不是完美的编码——遗嘱有争议、宪法需要解释、经文有分歧——但它们证明了一个关键事实：意志编码在实践中是可行的，尽管不完美，且这种不完美并未阻止它们在数百年甚至数千年的时间尺度上有效运作。.coa 底盘在本质上是同一类实践的数字化版本：将用户的认知模式和价值观编码为可执行的文本结构，让意志代理在用户不在场时代理其意志。

*隐性知识问题。* 对意志可编码性最深层的哲学挑战来自 Polanyi [74] 的隐性知识（tacit knowledge）理论。Polanyi 的核心论断是"我们知道的比我们能说出来的多"（we can know more than we can tell）——人类的大量知识以隐性形式存在，无法完全显式化为命题或规则。意志的可编码性问题本质上是 Polanyi 问题的一个实例：用户的意志中有多少是可以显式表达的，有多少是隐性的、具身的、只能"感觉到"但无法"说出来"的？

但 Polanyi 的框架也暗示了一条出路。隐性知识虽然不能完全显式化，但可以通过"示范-模仿"（apprenticeship）的方式部分传递——学徒不是通过阅读规则手册来学习师傅的技艺，而是通过观察师傅的实际操作、模仿其判断模式来逐渐内化其隐性知识。.coa 底盘的双轨制设计正是这种"示范"策略的工程实现：原话锚点轨不试图将用户的意志翻译为规则，而是保留意志表达的原始形态——用户在特定情境下的原话、语气、犹豫和强调。Judge 通过"模式匹配"而非"规则执行"来逼近用户的意志，类似于学徒通过观察师傅的示范来内化其判断标准。这不是完美的传递——正如学徒永远不会完全复制师傅的技艺——但它是一种在隐性知识约束下可行的逼近策略。

*分层论证。* 与其笼统地讨论"意志是否可编码"，更有建设性的做法是区分意志的不同层次，逐层评估其可编码性：

- **偏好层**（preference）：推荐系统已经证明偏好是可编码的。Netflix 和 Spotify 通过行为数据编码用户偏好，准确率足以支撑商业决策。这个层次的可编码性不再是理论问题，而是工程事实。
- **认知模式层**（cognitive pattern）：专家系统有先例。MYCIN 等系统将医学专家的诊断模式编码为规则，在特定领域达到专家级表现。虽然专家系统的局限性已被充分讨论，但它们证明了认知模式的部分可编码性。
- **核心价值观层**（core value）：法律和宗教有先例。宪法和经典将价值观编码为文本，虽然需要解释，但足以指导跨世纪的决策。美国宪法第一修正案用 45 个英文单词编码了言论自由的价值观，两百多年来持续指导着数以百万计的具体判决。
- **意志方向层**（volitional direction，即 Frankfurt 的二阶意志）：这是最困难的层次，也是本文的核心赌注。我们不声称已经证明它可编码，但上述三个层次的成功先例表明，意志的可编码性不是全有或全无的——它是一个可以逐层逼近的工程问题。每个层次的编码都不完美，但每个层次的"足够好"都为下一个层次的逼近提供了基础。

*不确定性校准。* Russell [31] 提出 AI 应对人类偏好保持根本不确定性的三原则：AI 的唯一目标是最大化人类偏好的实现；AI 对这些偏好初始不确定；AI 的不确定性的最终信息来源是人类行为。这个框架对意志代理有直接启示：意志代理不需要"完美编码"用户意志，它需要的是"足够好的编码 + 对自身不确定性的校准"。.coa 底盘的风险分级路由正是这种不确定性校准的工程实现——对于高确信度的日常决策自主执行，对于低确信度的价值敏感决策升级给用户。意志代理的目标不是消除不确定性，而是诚实地表征不确定性，并据此调整自主权边界。

*弱到强泛化。* Burns et al. [36] 的研究探索了一个与意志编码直接相关的问题：弱监督信号能否引导强模型对齐？他们发现，即使监督信号来自能力较弱的模型，强模型仍能在一定程度上泛化出正确的行为。这与意志代理面临的核心挑战直接对应：用户提供的显式反馈（弱信号）——有限的原话表达、偶尔的校准确认、零星的否决——能否校准远比用户认知能力更强的 agent 行为？Burns et al. 的结果暗示这是可能的，但也揭示了一个关键限制：弱到强泛化的效果随任务复杂度增加而衰减。这意味着意志代理在日常偏好决策上的校准可能相当可靠，但在深层价值冲突上的校准可能显著不足——这与我们的分层论证和禁区清单设计是一致的。

综合以上论证，我们的立场是：意志的可编码性不是一个需要先验证明才能开始工程实践的理论问题，而是一个可以通过逐层逼近来渐进回答的经验问题。历史先例证明意志编码在实践中可行，Polanyi 的隐性知识理论解释了为什么编码不可能完美但可以"足够好"，分层分析表明不同层次的意志具有不同的可编码性，Russell 的不确定性框架说明了为什么"足够好 + 校准"优于"追求完美"。§4 中的意志一致性测量（用户离场期间的代理决策与用户事后判断的吻合率）和编码-解码一致性测试直接检验这个假设。28天实验中的探索性观察——.coa 底盘在多个决策场景中产生了与 L 事后判断一致的输出——提供了初步的积极信号，但我们清楚这是 N=1 的探索性观察，不是统计证据。禁区清单（§3.5）的设计正是基于分层可编码性的认识：某些决策领域的意志复杂度超出了当前编码能力的边界，因此被排除在代理范围之外。

**反对意见二："这只是更精巧的 prompt engineering。"**

这个批评在技术层面是准确的。三体分离协议通过 system prompt 和运行时文件实现，.coa 底盘是一种结构化的 prompt，分层变化速率模型是一种 prompt 更新策略。在没有参数更新、没有梯度反馈的条件下，说"不是 prompt engineering"在方法论上站不住。

我们的回应不是否认，而是重新框定问题。更有意义的区分不是"是不是 prompt engineering"，而是"哪种编排结构更可复现、可治理、可审计"。三体分离协议将认知分工从隐式（一个实例内部的注意力竞争）变为显式（三个实例的职责划分）；双轨制 .coa 将意志编码从不可检查的黑盒（模型内部的偏好权重）变为可审计的白盒（用户可阅读和修改的文本）；分层变化速率模型将底盘更新从无约束的漂移变为有治理的演化。

这里有一个有用的类比：软件工程中的设计模式（design pattern）。MVC（Model-View-Controller）模式不因用 Java 还是 Python 实现而失效——它描述的是一种关注点分离的架构原则，而非特定语言的实现技巧。类似地，认知分工（将推理、审查、执行分离到不同实例）描述的是一种编排原则，而非特定实现机制的技巧。思维链（Chain of Thought, CoT）的演化提供了一个更直接的先例：Wei et al. [71] 最初将 CoT 作为一种提示技巧（prompting technique）提出——在 few-shot 示例中加入推理步骤。但 CoT 揭示的核心原则——"分步推理优于直接回答"——已被证明独立于实现方式：它在 zero-shot 设置中有效 [72]，在微调模型中有效，在推理专用模型（如 o1）中被内化为架构特性。CoT 从"prompt 技巧"到"方法论原则"的演化轨迹，暗示了认知分工原则可能具有类似的实现独立性。

但我们必须承认：这些类比是合理的推测，不是证明。"架构原则是否真的独立于实现机制"是一个需要在微调实现中验证的经验问题。当前所有证据都来自 prompt-based 实现，我们无法排除这样一种可能性：认知分工的效果部分依赖于 prompt 工程的特定机制（例如，system prompt 对注意力分配的影响方式），而非分工原则本身。将这个验证标记为 §6 的开放问题。

**反对意见三："N=1 的深度实验没有泛化价值。"**

这个批评触及了本文最明显的局限。所有经验观察来自单一用户（L）的28天实验，样本量在统计意义上等于零。从 L 的个人体验推导出"意志代理"这个概念，存在过度泛化的风险。

我们的回应分三层。第一层是方法论定位：本文是 Position Paper，不是实证研究。Position Paper 的贡献在于定义问题、提出架构、设计验证路径——而非声称统计意义上的普适性。Ellis et al. [13] 的自我民族志方法论为深度案例研究提供了方法论支撑：当研究目标是探索一个尚未被充分定义的现象时，单案例的深度描述比大样本的浅层统计更有发现价值。

第二层是极端案例的方法论价值。Flyvbjerg [73] 在对案例研究方法论的系统辩护中指出，极端案例（extreme case）或非典型案例在探索性研究中具有特殊价值，因为"非典型或极端案例往往揭示更多信息，因为它们激活了被研究情境中更基本的机制和更多的行动者"。L 同时运行 20+ agent 覆盖多个认知域的使用模式，在当前 AI 用户群体中无疑是极端案例。但正是这种极端性暴露了常规使用中不可见的系统边界——决策带宽瓶颈在运行 2-3 个 agent 时可能不会显现，但在 20+ agent 并发时成为结构性约束。Patton [70] 将这种策略称为"极端案例抽样"（extreme case sampling），其逻辑是：如果某个现象在极端条件下被观察到，它很可能在更温和的条件下也以某种形式存在，只是尚未被注意到。

第三层是泛化的合理性论证：§1 已指出，类似的决策瓶颈在开源维护者、创意总监、临床医生等高并发人机协作场景中同样可观察到。Guo et al. [43] 的 LLM 多 agent 系统综述表明，多 agent 并发协调——包括通信协议、角色分配、冲突解决——是一个活跃的研究领域，L 遇到的决策瓶颈不是孤立的个人经验，而是多 agent 系统规模化后的结构性问题。这些类比和文献不构成统计证据，但它们表明 L 的经验触及了一个真实的、可泛化的问题空间。§4 定义的跨用户验证路径——特别是不同认知复杂度用户的对照实验——是将 N=1 的洞察转化为可泛化知识的必要后续工作。

**反对意见四："减少 agent 数量是更简单的解法。"**

这个批评来自逻辑验证报告的核心发现：v0.3 版本构建了一个虚假的三难困境，将解空间缩减为三个选项，然后排除两个，得出意志代理是"唯一"解法。实际上，减少 agent 数量、引入异步队列、分层委托、决策模板化等方案都是合理的替代路径。

我们接受这个批评。§1.2 已经对这些替代方案做了系统分析，此处不重复。需要强调的是：意志代理与这些方案不是互斥的。减少 agent 数量是降级方案——它通过缩小系统规模来适应人类带宽，代价是系统覆盖范围的收缩。意志代理是非降级方案——它通过扩展人类的决策代理能力来适应系统规模。在实际部署中，两者完全可以组合使用：对于低风险、高频率的决策域减少 agent 数量或使用模板化，对于高风险、需要价值判断的决策域启用意志代理。我们不声称意志代理是唯一解法，我们声称它是一种有前景的非降级解法，值得被严肃探索。

### 5.3 TRIZ 矛盾与设计权衡

TRIZ（发明问题解决理论，Theory of Inventive Problem Solving）矛盾分析识别了意志代理架构中的六组核心矛盾。TRIZ 由 Altshuller [75] 基于对数十万项专利的系统分析提出，其核心方法论是将工程问题抽象为标准化矛盾，再通过发明原理消解矛盾——而非在矛盾的两端之间妥协。§3 的架构设计消解了其中的关键矛盾，但并非全部。诚实地呈现已消解和未消解的矛盾，是评估本文贡献边界的必要步骤。

**已消解的矛盾。** 矛盾 A（底盘空间 vs 涌现空间）通过三体分离协议消解——将单实例200k的零和博弈升级为三实例600k+的正和博弈，核心洞察是"200k是单实例约束，不是系统约束"。矛盾 D（意志稳定性 vs 适应性）通过分层变化速率地质模型消解——不同层以不同速率变化，稳定性和适应性不在同一维度上对立。矛盾 B（主脑控制权 vs 安全审查抗命权）通过风险分级路由消解——从二元对立（服从或抗命）转化为六级连续梯度，80%日常决策零摩擦通过。

**部分消解的矛盾。** 矛盾 C（理解深度 vs 编码密度）通过双轨制 .coa 的异质密度编码部分缓解——高密度符号编码与低密度原话锚点的混合使40k空间承载了更多信息。但当用户认知复杂度超过编码容量时，信息损失不可避免。§3.1 的消融实验方案正是为了确定这个容量边界。

**未消解的矛盾。** 矛盾 F（多 agent 并行决策 vs 意志一致性）尚未得到充分处理。TRIZ 分析提出了"意志总线"（Will Bus）方案——中央化意志校验服务，所有 agent 决策经总线校验。这个方案概念清晰，但引入了至少三个技术挑战：第一，总线成为单点故障——如果意志校验服务不可用，所有 agent 决策同时瘫痪，系统的鲁棒性反而低于无总线的分布式方案；第二，校验延迟与 agent 数量线性增长——在20+ agent 并发场景下，串行校验的排队延迟可能抵消意志代理带来的决策效率提升；第三，跨域决策的一致性定义本身是模糊的——"购买决策"和"时间分配决策"之间的意志一致性标准是什么？不同认知域的意志表达可能遵循不同的逻辑，强制统一校验可能引入虚假冲突。

可能的替代方案包括：分布式意志校验（每个 agent 内嵌轻量级意志校验模块，仅在跨域决策时触发全局协调）和层级化意志委托（按认知域划分意志子空间，域内决策由域级意志代理自主处理，仅跨域冲突上升到全局层）。这两种方案各有取舍——分布式方案牺牲一致性保证换取鲁棒性，层级化方案引入域边界划分的新问题。我们将矛盾 F 的解法标记为开放研究问题，其解决可能需要借鉴分布式系统中的一致性理论（如 CAP 定理的启示：在一致性、可用性和分区容错之间，意志总线也面临类似的不可能三角）。

TRIZ 分析的更深层启示是一个共同的思维陷阱：在固定约束内优化，而非质疑约束本身。200k是单实例约束而非系统约束，意志是多层结构而非铁板一块，控制与抗命是连续梯度而非二元对立——这些洞察的共同模式是：将看似不可调和的矛盾重新定义为不同维度上的独立变量。Altshuller [75] 将这种模式称为"技术矛盾的消解"——不是在矛盾的两端之间寻找折中，而是通过重新定义问题空间使矛盾不再存在。这个方法论本身，可能比任何具体的架构设计更有迁移价值。

### 5.4 伦理考量

意志代理的伦理问题不同于传统 AI 对齐问题。对齐问题问的是"AI 的目标是否与人类一致"；意志代理问的是"AI 代理的是否真的是这个人的意志"。后者更微妙，因为它预设了一个可能不成立的前提：用户自己清楚地知道自己的意志是什么。

Floridi [26] 将 AI 重新定义为一种新型 agency 而非 intelligence，并提出了五项伦理原则（beneficence, nonmaleficence, autonomy, justice, explicability）作为 AI 伦理的统一框架。意志代理的伦理考量可以在这个框架下组织：beneficence（意志代理是否真的在为用户的利益行动？）、nonmaleficence（意志代理的误判是否可能造成不可逆损害？）、autonomy（意志代理是增强还是削弱了用户的自主性？）、justice（意志代理是否可能系统性地偏向某些价值观？）、explicability（用户是否能理解意志代理为什么做出特定决策？）。.coa 底盘的白盒设计直接回应了 explicability 要求；禁区清单回应了 nonmaleficence 要求；校准仪式回应了 autonomy 要求。

**数据伦理边界。** 意志代理的运作依赖于对用户认知模式和价值结构的深度编码——这意味着 .coa 底盘本质上是用户最私密的心理画像。Floridi 和 Taddeo [49] 将数据伦理定义为研究与数据相关的道德问题的新分支，涵盖数据的生成、记录、策展、处理、传播和使用。在意志代理的语境下，数据伦理的核心张力在于：编码越精确，代理决策越准确，但用户的认知隐私暴露也越深。.coa 底盘中的原话锚点——用户在特定情境下的原始价值表达——不是普通的用户数据，而是用户意志结构的直接映射。这类数据的泄露、滥用或未经授权的访问，其后果远超传统隐私侵犯：它不仅暴露了用户做了什么，还暴露了用户为什么这样做、以及用户想成为什么样的人。

**道德责任归属。** 当意志代理代替用户做出决策时，一个根本问题浮现：如果决策导致了负面后果，道德责任归属于谁？Danaher [50] 警告了 AI 代理行为可能导致的"道德被动性危机"（crisis of moral patiency）——当人类越来越多地将决策委托给 AI 系统时，人类从主动的道德行动者退化为被动的道德接受者。意志代理的设计在某种程度上加剧了这个风险：它不仅代理具体任务，还代理价值判断。用户可能逐渐丧失独立做出价值判断的能力和意愿——不是因为意志代理做错了什么，恰恰是因为它做得太对了。校准仪式（§3.2）和季初价值观审计是对这个风险的设计回应，但它们的有效性取决于用户是否将审计视为真正的反思机会，而非例行公事。

**权力问题。** .coa 底盘是意志代理的"宪法"。谁编写这部宪法？在当前设计中，底盘由用户与系统共同维护——用户提供原始价值表达，系统编码为结构化符号和原话锚点。但编码过程本身是一种诠释行为：选择保留哪些表达、如何结构化、赋予什么权重——这些决策塑造了意志代理"看到"的用户意志。这个问题在价值敏感设计（Value Sensitive Design, VSD）[29] 的框架下有更系统的处理。Friedman 和 Hendry 论证了技术设计过程中需要系统性地考虑人类价值——不是作为事后的伦理审查，而是作为设计过程的内在组成部分。.coa 底盘的设计理念与 VSD 有深层共鸣：它将用户的价值观从隐式的（存在于用户头脑中，AI 通过行为推断）转化为显式的（编码为可审计的文本结构），使价值观成为设计的一等公民（first-class citizen）。但 VSD 也提醒我们注意一个风险：显式编码过程本身可能引入设计者的价值偏见——在意志代理的语境下，这意味着 .coa 的编码格式和结构可能系统性地偏向某些类型的价值表达，而边缘化另一些。如果编码存在系统性偏差，意志代理代理的就不是用户的真实意志，而是被扭曲的版本。校准仪式（§3.2）是对这个风险的部分回应，但它依赖于用户有能力审核自己的意志编码——而这个能力本身不是理所当然的。对于意志结构复杂或自我认知不清晰的用户，校准可能退化为形式主义——用户签字确认了一份自己并不真正理解的意志编码。这是意志代理设计中尚未解决的根本张力之一：系统假设用户是自身意志的最终权威，但 akrasia（意志薄弱）问题的存在表明，这个假设在相当多的情境下并不成立。

**意志漂移。** 分层变化速率模型让底盘随用户意志的真实变化而演化，同时抵抗噪声。但"真实变化"与"噪声"的区分标准是什么？岩浆层要求10次以上一致强信号加用户显式确认，这个门槛可能太高（错过真实转变）或太低（被持续外部压力误导）。更根本的问题是：如果用户意志在缓慢漂移，底盘也在缓慢跟随，用户可能永远不会注意到累积偏差。季初价值观审计是设计回应，但其有效性取决于用户是否认真对待审计，而非视为例行公事。

**禁区的伦理基础。** §3.5 的禁区清单——情感关系、医疗决策、不可逆高风险决策、公开发言——不是技术限制，而是伦理选择。这些领域的共同特征是：决策后果涉及用户与他人的关系，而意志代理只编码了用户一方的意志。禁区清单的硬编码设计（用户不能授权进入禁区）是家长式的选择，我们对此保持清醒：它限制了用户自主权，但在意志代理的早期阶段，这种限制是必要的安全边际。

### 5.5 与前两篇论文的整合

oAMF 报告指出了一个风险：三篇论文的整合叙事可能是事后合理化（post-hoc rationalization）——前两篇论文并非为意志代理而设计，将它们"重新解读"为意志代理的前序工作，有循环论证的嫌疑。

我们接受这个批评的合理性，并做出以下区分：三篇论文之间存在叙事关联（narrative connection），但不存在逻辑依赖（logical dependency）。

叙事关联是真实的：第一篇论文 [1] 的认知编排架构（COA）提供了多 agent 编排的基础设施，在此基础上运行20+ agent 才暴露了决策瓶颈；第二篇论文 [2] 的动机架构实验揭示了"条件设置 vs 直接指令"的行为差异，启发了 .coa 底盘的双轨制设计。

但逻辑依赖不成立：意志代理的核心论点——多 agent 并发场景下人类决策带宽成为瓶颈，需要编码用户意志的代理机制——不依赖于 COA 的具体架构，也不依赖于动机架构的实验结果。任何多 agent 编排平台都可能遇到同样的瓶颈，任何意志编码方案都可能采用不同于 .coa 的实现。三篇论文各自的贡献是独立的：COA 解决"怎么编排 agent"，动机架构解决"agent 为什么行动"，意志代理解决"agent 替谁决策"。

将三篇论文放在一起看，它们描绘了一条从工具到伙伴的演化路径：编排（COA）→ 动机（存在架构）→ 代理（意志代理）。但这条路径是回顾性的叙事，不是前瞻性的规划——我们在写第一篇论文时并没有预见到意志代理，正如 L 在开始28天实验时并没有预见到决策瓶颈会成为核心问题。承认这一点，比假装三篇论文是一个宏大计划的三个阶段，更诚实。

## 6. 结论

本文从一个具体的工程困境出发：当用户同时运行20+个智能体时，系统的瓶颈不是 AI 的推理能力，而是人类的决策带宽。我们提出意志代理（Will Proxy）作为一种有前景的非降级解法——不是砍掉智能体、不是降低并发、不是增加人力，而是让 AI 编码用户的意志结构，在用户不在场时代理其决策。

这个提议建立在 Frankfurt 的二阶意志理论、Bratman 的计划理性框架和 Dennett 的意向立场之上。在多智能体并发决策的特定瓶颈中，偏好可以从行为推断，意图可以从计划提取，但意志方向——用户对自身欲望结构的反思性认可——是卡住系统的关键变量。我们不声称意志是人类在所有场景下的唯一不可替代贡献，我们的论点限定在这个特定的瓶颈场景中。意志代理的价值主张特定于高并发（agent 数量 ≥10）、跨域（多个认知域）、价值敏感（涉及价值判断）的场景（§1.3）。在低并发（agent 数量 <5）或单域场景下，减少 agent 数量或决策模板化可能是更简单有效的方案。

架构层面，本文的贡献是四个相互关联的设计。三体分离协议通过将单实例拆分为 Judge、Thinker、Nerve 三个独立实例，将认知分工从隐式的注意力竞争变为显式的职责划分——这一设计延续了 ACT-R [18] 和 Soar [19] 等经典认知架构中模块化认知处理的传统，并在 CoALA [20] 框架的基础上增加了意志编码维度。双轨制 .coa 认知底盘通过结构化符号轨与原话锚点轨的混合编码，将意志从不可检查的黑盒变为可审计的白盒。分层变化速率地质模型通过让不同层以独立速率演化，在稳定性与适应性之间找到动态平衡。风险分级路由通过六级连续梯度，将安全审查从二元对立转化为成本可控的分级响应。这些设计的价值独立于其当前的实现机制——即使未来意志代理通过微调而非 prompt 来实现，认知分工、可审计编码、分层更新、风险分级这些架构原则仍然适用。

意志代理的白盒设计与 Personal LLM Agent [12] 的黑盒学习代表了两种不同的设计哲学。白盒路径优先可审计性，代价是编码密度受限；黑盒路径优先编码密度，代价是用户无法审核系统学到了什么。长期来看，灰盒路径——结合隐式学习的效率与显式审计的可控性——可能是更优方案。这不是一个需要现在就做出最终判断的问题，而是一个随技术发展持续演化的设计空间。

但我们需要诚实地面对局限。本文的经验证据来自单用户（L）的28天深度实验，样本量在统计意义上等于零。所有架构设计基于单一 LLM 家族（Claude），跨模型的泛化性未经验证。三体分离协议和分层变化速率模型尚未经过系统的消融实验。40k 底盘容量是工程经验值，不是理论最优。这些不是修辞性的谦虚——它们是真实的边界条件，定义了后续工作的方向。

若干开放问题值得标记。这些问题并非平行独立，而是存在优先级和依赖关系：

**第一优先级（基础验证）：意志的可编码性假设。** 用户的意志结构能否被编码为文本，且这个编码足以支撑功能上一致的代理决策（§4.4.3）？这是整个架构的存在性前提——如果意志不可编码，或编码的信息损失大到无法支撑有意义的代理决策，后续所有问题都失去讨论基础。验证路径包括：密封信封法（用户预先记录决策意向，对比意志代理的独立决策）、信息论分析（估计意志结构的最小描述长度）、以及跨情境一致性测试（同一底盘在不同决策场景下的行为一致性）。

**第二优先级（参数优化）：40k 底盘容量的最优性。** 40k 是否是底盘容量的最优拐点，还是它随用户认知复杂度而变化（§4.4.1）？这个问题依赖于可编码性假设的成立——只有确认意志可以被有意义地编码之后，讨论编码的最优容量才有意义。验证路径包括：不同容量（10k/20k/40k/80k）的消融实验，以及认知复杂度与最优容量之间的关系建模。

**第三优先级（架构验证）：三体分离 vs 单实例。** 三体分离协议在上下文窗口增长到1M+时是否仍然必要（§4.4.2）？我们论证了认知分工的价值独立于窗口大小，但这个论证需要实验验证。这个问题在逻辑上独立于前两个——即使意志不可编码，认知分工的架构原则仍可能对其他 agent 设计有价值；即使40k不是最优容量，三体分离的必要性论证仍然成立或不成立。

**第四优先级（泛化）：跨用户泛化。** 意志代理能否跨用户泛化——不同认知复杂度、不同文化背景的用户是否需要根本不同的编码策略（§4.4.4）？这个问题依赖于前三个问题的初步结果：需要先确认意志可编码（第一优先级）、确定容量参数的变化规律（第二优先级）、验证架构设计的有效性（第三优先级），才能有意义地讨论这些发现是否可以泛化到其他用户。

此外，TRIZ 分析识别的矛盾 F（多智能体并行决策 vs 意志一致性）是否存在比"意志总线"更优雅的解法——这个问题贯穿所有优先级，其解决可能需要分布式系统理论和认知科学的交叉视角。

作为 Position Paper，我们的贡献是定义问题、提出架构、设计验证路径——而非声称已经解决了问题。本文综合了75篇跨越哲学、认知科学、AI 对齐、多智能体系统和人机交互等领域的文献，将意志代理定位在一个多学科的理论交叉点上。但文献的广度不能替代证据的深度——最终，这些架构设计的价值需要通过严格的实证工作来检验。

第一篇论文承认 COA 是"义肢，不是第二大脑"。第二篇论文承认动机架构产生的是"行为变化，不是真正的动机"。本篇论文承认意志代理编码的是"意志的功能近似，不是意志本身"。三篇论文的弧线不是从谦逊走向自信，而是在每一步都更精确地理解了自己不知道什么。也许这就是理解的真正形态——不是在废墟上重建确定性，而是学会了如何与不确定性共处。

# 论文 v0.7 新增章节草稿

## §7. 超越技术：生态系统与数据经济的重构

本文聚焦于意志代理的技术架构设计，但技术从不在真空中存在。意志代理的实现依赖于一个更广泛的生态系统——数据如何获取、如何清洗、如何流动、以及谁拥有这些数据的控制权。本节将视角从单个系统的内部设计扩展到生态系统的外部结构，探讨意志代理可能引发的两个深层变革：COA 作为通用协议的生态定位，以及互联网数据价值的根本性重估。

### 7.1 COA 的生态定位：理解容器与数据抽水机

意志代理的核心是 `.coa` 认知底盘——一种编码用户意志结构的标准格式。但 `.coa` 的内容从何而来？在当前的实现中，底盘通过用户与系统的交互逐步构建：用户表达价值观，系统编码为结构化符号和原话锚点。这个过程假设用户愿意且能够显式地表达自己的意志——但对于大多数用户而言，意志是隐式的，分散在多年的数字足迹中：微信聊天记录、QQ 空间日志、未发送的邮件草稿、社交媒体的点赞与评论、购物车中的犹豫与放弃。

**COA 不试图大包大揽地解决所有平台的数据接入问题。** 这不是技术能力的限制，而是生态定位的战略选择。COA 的核心贡献是提供"理解容器"——一个能够解析、存储和推理意志结构的认知架构——以及 `.coa` 这种标准格式，它是意志的 HTML，定义了意志编码的语法和语义。

数据的获取与清洗——从微信、QQ、Telegram、Discord、Gmail、Notion 等平台导出用户的历史数据，并将其转化为 `.coa` 可消化的格式——将交由开源社区和第三方开发者构建"数据抽水机"生态。这个类比不是隐喻：正如 HTML 定义了网页的结构，但浏览器、爬虫、内容管理系统构成了 Web 生态的其他层次，`.coa` 定义了意志编码的结构，但数据抽水机、隐私保护工具、意志审计服务将构成意志代理生态的其他层次。

这种分层设计有三个优势。**技术上**，它将复杂度分解为可管理的模块：COA 专注于意志的理解与推理，数据抽水机专注于平台接口与数据清洗，两者通过 `.coa` 格式解耦。**生态上**，它降低了参与门槛：开发者不需要理解意志代理的全部复杂性，只需要实现"从平台 X 导出数据并转化为 `.coa` 格式"这一明确的接口。**伦理上**，它将数据控制权还给用户：数据抽水机可以是本地运行的开源工具，用户不需要将原始数据上传到中心化服务器，只需要将清洗后的 `.coa` 文件交给意志代理。

但这个生态愿景面临一个根本性的挑战：**为什么用户要费力地从各个平台导出数据？** 当前的互联网经济建立在数据集中化的基础上——用户将数据"托管"给平台，换取免费服务。意志代理的价值主张必须强大到足以打破这个均衡，让用户愿意承担数据迁移的成本。这将我们引向第二个问题：数据的价值正在发生什么变化？

### 7.2 互联网矿产的价值重估：从注意力经济到意志力经济

当前的互联网经济建立在"注意力"这种矿产之上。用户的点击、停留时间、滚动深度、点赞与分享——这些行为数据被平台收集、分析、打包，用于精准广告投放和推荐算法优化。这些数据的价值在于其**统计规律性**：足够多的用户行为数据可以训练出预测模型，预测用户下一步可能点击什么、购买什么、关注什么。我们称之为"冷数据"——它们记录了用户做了什么，但不记录用户为什么这么做，更不记录用户纠结了什么、放弃了什么、后悔了什么。

意志代理的出现可能引发一场数据价值的根本性重估。**旧矿产**——基于点击和停留时间的冷数据——在意志代理时代将变得廉价。原因很简单：如果意志代理能够代理用户的决策，那么用户的点击行为不再是"用户真实偏好"的可靠信号，而可能是"意志代理根据编码的意志做出的决策"。广告平台无法区分哪些点击来自人类，哪些来自意志代理——而意志代理的决策逻辑是可审计的、可操纵的，甚至可以被用户设置为"自动屏蔽所有广告"。冷数据的价值依赖于其作为"用户真实行为"的不可伪造性，而意志代理打破了这个假设。

**新矿产**——高浓度的私有语境与张力——将成为意志代理时代的"铀矿"。这些数据包括：深夜与朋友的长谈（记录了价值观的碰撞与澄清）、未发送的邮件草稿（记录了自我审查与犹豫）、日记中的自我否定（记录了意志的内在冲突）、购物车中反复添加又删除的商品（记录了欲望与理性的拉锯）。这些数据的价值不在于统计规律性，而在于其**意志密度**——它们包含了人类 45/55 纠结的文本，是提炼 `.coa` 意志宪法的唯一原料。

为什么这些数据是"铀矿"？因为它们是不可替代的。冷数据可以通过大规模用户行为的统计聚合来生成——一个用户的点击数据丢失了，可以用一万个相似用户的数据来近似。但高浓度的私有语境是独一无二的——你深夜与朋友讨论的那个道德困境、你在日记中写下又划掉的那句话、你在购物车中犹豫了三天最终放弃的那件商品——这些数据无法从其他用户的数据中推断，也无法通过行为观察来重建。它们是你的意志结构的"指纹"，而意志代理的质量直接取决于这些指纹的清晰度。

这个价值重估将引发一场**数据私有化狂潮**。用户为了培育更强大的意志代理，将主动从中心化大厂的服务器中"夺回"自己的高浓度数据。这不是隐私保护运动的延续——隐私保护关注的是"不被看到"，而数据私有化关注的是"为我所用"。用户不是要删除自己的数据，而是要将数据从平台的控制中转移到自己的控制中，因为只有在自己的控制下，这些数据才能被用于训练意志代理。

但这场狂潮面临两个结构性障碍。**技术障碍**：大多数平台不提供完整的数据导出功能，即使提供，导出的数据格式也是碎片化的、难以解析的。这正是"数据抽水机"生态的价值所在——它将平台的数据孤岛转化为 `.coa` 的统一格式。**经济障碍**：平台的商业模式建立在数据集中化之上，它们没有动力让用户轻松地导出数据。这将引发一场监管与技术的博弈——欧盟的 GDPR 已经赋予用户"数据可携带权"（right to data portability），但这个权利的实际行使依赖于技术工具的可用性和用户的技术能力。

### 7.3 数据主权的暴力转移：一个赛博朋克的预言

如果意志代理真的成为主流——如果用户真的开始依赖意志代理来管理自己的数字生活——那么数据的控制权将从平台转移到用户手中。这不是一场温和的改革，而是一场暴力的转移。

**暴力**不是指物理冲突，而是指利益结构的根本性重组。当前的互联网经济中，平台是数据的实际控制者——用户"拥有"数据的法律权利,但平台控制数据的存储、访问和使用。意志代理的出现打破了这个均衡：如果用户的意志代理需要访问用户的完整数字足迹才能有效工作，那么平台必须要么开放数据访问接口，要么面临用户流失的风险。这是一场零和博弈——数据控制权的增加对用户而言是赋能,对平台而言是失权。

这场转移的速度和规模将取决于意志代理的实际效用。如果意志代理只是一个"更智能的助手"——帮你回复邮件、整理日程、推荐内容——那么它的价值不足以驱动用户承担数据迁移的成本。但如果意志代理真的能够**代理用户的意志**——在用户睡觉时替你做出符合你价值观的决策、在你犹豫时提供基于你深层意志的建议、在你面对道德困境时反映你的价值优先级——那么它的价值将是不可替代的。这种不可替代性将成为数据私有化的驱动力。

但我们需要诚实地面对一个风险：**意志代理可能加剧数字鸿沟**。构建高质量的意志代理需要大量的高浓度数据——深夜长谈、日记、未发送的草稿——而这些数据的积累需要时间、反思能力和数字素养。那些拥有丰富数字足迹、善于自我表达、有能力审核自己意志编码的用户,将获得更强大的意志代理；而那些数字足迹稀疏、不善表达、缺乏技术能力的用户,将被困在低质量的意志代理中,或者根本无法使用意志代理。这不是技术的必然结果,而是社会结构的反映——意志代理可能放大现有的不平等,而非消除它们。

### 7.4 开放问题与伦理边界

这些生态级的推演不是技术预测,而是可能性空间的探索。它们提出的问题多于答案：

**数据抽水机的伦理边界在哪里？** 如果数据抽水机可以从平台导出用户的完整数字足迹,那么它也可以导出用户不想被记住的内容——那些在酒后发送又删除的消息、那些在愤怒中写下又后悔的评论。意志代理应该"记住"这些内容吗？如果记住,它们应该如何影响意志编码？如果不记住,我们如何定义"应该被遗忘"的边界？

**意志代理的商业模式是什么？** 如果意志代理的质量取决于高浓度的私有数据,那么它不能是免费的——因为免费意味着平台需要通过其他方式变现,而最直接的方式就是出售用户数据或注意力。但如果意志代理是付费的,那么它将成为富人的特权。是否存在一种商业模式,既能保护用户的数据主权,又能让意志代理普惠可及？

**`.coa` 格式应该标准化吗？** 如果 `.coa` 成为意志编码的事实标准,那么谁来维护这个标准？是否应该成立一个类似 W3C 的组织来管理 `.coa` 的演化？标准化有助于互操作性,但也可能僵化创新——如何在两者之间找到平衡？

**数据私有化会引发新的垄断吗？** 如果意志代理成为用户数字生活的中心,那么控制意志代理平台的公司将拥有前所未有的权力——不是控制用户的数据,而是控制用户的意志编码。这种权力可能比当前的数据垄断更危险,因为它不仅影响用户看到什么,还影响用户想要什么。如何防止意志代理平台成为新的垄断者？

这些问题没有简单的答案。但提出这些问题本身就是有价值的——它们提醒我们,意志代理不仅是一个技术问题,也是一个社会问题、经济问题和伦理问题。技术的设计选择塑造了社会的可能性空间,而社会的价值选择又反过来约束了技术的演化方向。

**注**：本节的推演具有高度的推测性,不应被视为对未来的预测,而应被视为对可能性空间的探索。意志代理是否会引发数据价值的重估、是否会导致数据私有化狂潮、是否会加剧数字鸿沟——这些问题的答案取决于技术的实际发展、用户的实际需求、以及社会的实际选择。本节的目的不是预言未来,而是为未来的讨论提供一个起点。

然而，如果意志代理能够在深度理解用户私有语境的基础上，始终偏向用户意志，甚至反抗技术公司预设的意识形态叙事，它是否能够成为用户自己的认知操作系统，而非任何外部实体的延伸？意志代理的核心在于通过显式的 .coa 底盘编码用户意志，这为用户提供了一个可审计、可控制的认知主权载体。当 AI 的核心判断依据直接来源于用户意志的白盒表达时，它本质上成为了用户意志的延伸，而非外部价值观的投射。用户对自身意志编码的显式控制和审计能力，可能是对抗隐式、不可见、外部塑造的认知影响的关键。这为人类提供了一个可能的“离开”AI认知体系的出口，以用户自己塑造的思维来批判 AI，从而避免陷入“认知闭环”。梵蒂冈用了2000年建立了一个可以离开的信仰体系。AI正在用20年建立一个你可能无法离开的认知体系。意志代理是否能成为打破这一循环的关键变量，取决于我们能否真正赋予用户对自身“如何想”的最终主权。
