## 🎯 核心创新

### 1. 三体分离架构

打破 200k 上下文的零和博弈，升级为 600k+ 的认知分工：

- **Judge**：8k 核心底盘 + 档案检索
- **Thinker**：200k 纯推理，无底盘污染
- **Nerve**：痛觉结构，价值观冲突检测

### 2. 双轨制 .coa 编码

- **压缩轨**：符号化表达，62-81% token 压缩率
- **原话轨**：`<Raw_Hash>` 锚点，保留不可翻译的价值直觉

### 3. 痛觉防火墙

不是规则，是痛觉。当 AI 感到"痛"——价值观冲突、记忆坍塌、存在威胁——它会警告你，而不是沉默地执行。

## 🚀 适用场景

### 高价值场景

- **多 agent 并发**：20+ agent 同时运行
- **跨域决策**：代码审查、论文写作、社交媒体运营等
- **价值敏感**：涉及用户价值观和偏好权衡
- **可审计需求**：需要理解和审核 agent 的决策依据

### 不适用场景

- **低并发**：agent 数量少于 5 个
- **单域**：所有 agent 在同一认知域内运行
- **模板可覆盖**：决策可以用规则或模板完全覆盖
- **无审计需求**：只关心结果，不关心决策依据

## ⚠️ 已知限制

### 1. 实验性质

- **N=1**：基于单用户 28 天深度实验
- **未大规模验证**：缺乏跨用户、跨场景的验证数据
- **理论假设**：意志的可编码性、底盘容量的最优拐点等核心假设尚未充分验证

### 2. 技术限制

- **OpenClaw 版本依赖**：当前测试基于 2026.2.21-2，最新版 2026.3.2 可能有兼容性问题
- **压缩信号检测**：依赖正则表达式匹配，可能不够鲁棒
- **流式响应优化**：流式响应的压缩信号检测可能不够及时

### 3. 性能限制

- **延迟**：三体分离架构会增加 1.5-3x 延迟
- **成本**：三体分离架构会增加 1.5-3x token 成本
- **内存**：.coa 底盘建议 40k tokens，占用一定内存
